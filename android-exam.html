שאלת מבחן ב-Android — EditText → TextView

שאלת אנדרואיד קלאסית, פתורה מקצה לקצה עם הסברים. ואז ההרחבה לשני מסכים (Intent), ולבסוף טבלת הכללות — איך הקוד משתנה אם השאלה הייתה מבקשת משהו אחר.

📋
השאלה (בנוסח חמי) כתבו קוד אנדרואיד תקני ומלא, עם חלוקה מסודרת לקבצים, המציג אפליקציה בעלת מסך יחיד עם תיבת עריכת טקסט (EditText), כפתור (Button), ושדה הצגת טקסט (TextView). בעת לחיצה על הכפתור, הטקסט שבתיבת העריכה יוצג בשדה ההצגה. הוסיפו הסברים מתאימים.
📑 תוכן העניינים
  1. 1️⃣ הפתרון — מסך יחיד
  2. 2️⃣ הדפוס שמכליל (לשינון)
  3. 3️⃣ הרחבה — שני מסכים (Intent)
  4. 4️⃣ הכללות — איך זה משתנה אם השאלה אחרת

1️⃣ הפתרון — מסך יחיד

שלושה קבצים בלבד: העיצוב (XML), הלוגיקה (Java), והרישום (Manifest).

מבנה הקבצים (3)
res/layout/activity_main.xml   ← העיצוב
MainActivity.java              ← הלוגיקה
AndroidManifest.xml            ← רישום המסך

1. res/layout/activity_main.xml

xml · העיצוב
<LinearLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">

    <EditText
        android:id="@+id/etInput"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:hint="Enter text" />

    <Button
        android:id="@+id/btnUpdate"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Update" />

    <TextView
        android:id="@+id/tvOutput"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content" />

</LinearLayout>
💡
הסבר מיכל LinearLayout אנכי, ובתוכו 3 רכיבים. לכל רכיב layout_width/height ו-@+id (כי נוגעים בכולם מ-Java). EditText = קלט, Button = כפתור, TextView = הצגה (ריק — יתמלא בקוד).

2. MainActivity.java

java · הלוגיקה
public class MainActivity extends AppCompatActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        Button btn = findViewById(R.id.btnUpdate);
        btn.setOnClickListener(view -> {
            EditText et = findViewById(R.id.etInput);
            TextView tv = findViewById(R.id.tvOutput);
            tv.setText(et.getText().toString());
        });
    }
}
💡
הסבר onCreate רץ כשהמסך נולד → setContentView טוען את ה-XML → findViewById תופס את הכפתור → setOnClickListener מחווט לחיצה → בפנים: קח טקסט מ-et (getText().toString()) ושים ב-tv (setText).

3. AndroidManifest.xml

xml · הרישום
<activity android:name=".MainActivity">
    <intent-filter>
        <action   android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LAUNCHER" />
    </intent-filter>
</activity>
💡
הסבר רושם את המסך. MAIN + LAUNCHER = מסך הכניסה (נפתח בלחיצה על אייקון האפליקציה).

2️⃣ הדפוס שמכליל (לשינון)

🔑
שלושת השלבים — נכון לכל שאלת אנדרואיד XML מצייר (מיכל + רכיבים עם id) ← Java תופס (findViewById) ומחווט (setOnClickListener) ← Manifest רושם.
🏷️
על השמות — נקודה שמבלבלת הרבה השמות (id) הם שרירותיים — אתה בוחר. etInput/btnUpdate/tvOutput זה סתם שמות; יכולת לקרוא להם box1/b/result. הכלל היחיד: ה-id ב-XML (@+id/X) חייב להיות אותו שם כמו ב-Java (R.id.X) — זה הקישור. ה-et/btn/tv זו רק מוסכמה נחמדה (et=EditText, btn=Button), לא חובה.

3️⃣ הרחבה — שני מסכים (Intent)

התרחיש: מסך 1 — תיבת קלט + כפתור. בלחיצה, עוברים למסך 2 שמציג את הטקסט שהוקלד.

מבנה הקבצים (עכשיו כפול — 2 מסכים)
res/layout/activity_main.xml     ← עיצוב מסך 1
res/layout/activity_second.xml   ← עיצוב מסך 2
MainActivity.java                ← לוגיקה מסך 1
SecondActivity.java              ← לוגיקה מסך 2
AndroidManifest.xml              ← רושם את שני המסכים

1. activity_main.xml — מסך 1 (קלט + כפתור)

xml · מסך 1
<LinearLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">

    <EditText
        android:id="@+id/etInput"
        android:layout_width="match_parent"
        android:layout_height="wrap_content" />

    <Button
        android:id="@+id/btnGo"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Go" />
</LinearLayout>

2. MainActivity.java — בלחיצה: Intent + putExtra

java · מסך 1
public class MainActivity extends AppCompatActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        Button btn = findViewById(R.id.btnGo);
        btn.setOnClickListener(view -> {
            EditText et = findViewById(R.id.etInput);
            Intent intent = new Intent(this, SecondActivity.class);   // לאן עוברים
            intent.putExtra("text", et.getText().toString());          // צרף נתון
            startActivity(intent);                                     // בצע מעבר
        });
    }
}
💡
הסבר בדיוק כמו מסך יחיד, אבל בתוך הלחיצה — במקום setText, יוצרים Intent ("מהמסך הזה → אל SecondActivity"), מצרפים את הטקסט עם putExtra (מפתח "text"), ו-startActivity מבצע את המעבר.

3. activity_second.xml — מסך 2 (הצגה)

xml · מסך 2
<LinearLayout

    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical">

    <TextView
        android:id="@+id/tvOutput"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content" />
</LinearLayout>

4. SecondActivity.java — מקבל את הנתון ומציג

java · מסך 2
public class SecondActivity extends AppCompatActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_second);

        TextView tv = findViewById(R.id.tvOutput);
        Intent intent = getIntent();                              // ה-Intent שפתח אותי
        String text = intent.getExtras().getString("text");      // שלוף לפי המפתח
        tv.setText(text);
    }
}
💡
הסבר מסך 2 הוא Activity רגיל (onCreate + setContentView). אחר כך: getIntent() מחזיר את ה-Intent שפתח אותו, ו-getExtras().getString("text") שולף את הנתון לפי אותו מפתח ששלחנו. ואז מציגים.

5. AndroidManifest.xml — שני המסכים

xml · הרישום
<activity android:name=".MainActivity">
    <intent-filter>
        <action   android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LAUNCHER" />
    </intent-filter>
</activity>

<activity android:name=".SecondActivity" />
💡
הסבר MainActivity = מסך הכניסה (MAIN+LAUNCHER). SecondActivity = שורה אחת (מסך רגיל שמגיעים אליו ב-Intent).
➕
מה התווסף מהמסך היחיד? (זה כל ההבדל)
  • עוד layout + עוד Activity (מסך 2).
  • בלחיצה: Intent + putExtra + startActivity (במקום setText).
  • במסך 2: getIntent + getExtras לקבל את הנתון.
  • Manifest רושם את שניהם (השני בשורה אחת).
הדפוס לזכור: שולח = new Intent(this, X.class) + putExtra + startActivity. מקבל = getIntent().getExtras().getString(key).

4️⃣ הכללות — איך זה משתנה אם השאלה אחרת

הקסם: ה-שלד קבוע. כמעט תמיד משנים רק (א) אילו רכיבים יש ב-XML, ו-(ב) את השורה(ות) בתוך ה-setOnClickListener. הנה וריאציות נפוצות שחמי יכול לבקש — לכל אחת מסומן מה משתנה ב-XML ומה ב-Java:

א. "הצג את הטקסט באותיות גדולות"

XML ללא שינוי.

JAVA רק מוסיפים .toUpperCase() בשורת ההצגה:

java
tv.setText(et.getText().toString().toUpperCase());
ב. "חבר שני מספרים משתי תיבות והצג את הסכום"

XML שתי EditText (עם inputType="number") במקום אחת:

xml
<EditText android:id="@+id/etA" ... android:inputType="number" />
<EditText android:id="@+id/etB" ... android:inputType="number" />

JAVA ממירים למספר (Integer.parseInt), מחברים, ומציגים:

java
EditText etA = findViewById(R.id.etA);
EditText etB = findViewById(R.id.etB);
int sum = Integer.parseInt(etA.getText().toString())
        + Integer.parseInt(etB.getText().toString());
tv.setText(String.valueOf(sum));
ג. "הוסף כפתור 'נקה' שמרוקן את התיבה"

XML עוד Button:

xml
<Button android:id="@+id/btnClear" ... android:text="Clear" />

JAVA עוד setOnClickListener, ובפנים שורה אחת:

java
Button btnClear = findViewById(R.id.btnClear);
btnClear.setOnClickListener(v -> {
    et.setText("");      // מרוקן את התיבה
    tv.setText("");      // ומנקה את ההצגה
});
ד. "הצג את הטקסט רק אם תיבת סימון (CheckBox) מסומנת"

XML מוסיפים CheckBox:

xml
<CheckBox android:id="@+id/cbShow" ... android:text="Show?" />

JAVA בודקים isChecked() בתוך הלחיצה:

java
CheckBox cb = findViewById(R.id.cbShow);
if (cb.isChecked()) {
    tv.setText(et.getText().toString());
}
ה. "בחירה מתוך כמה אפשרויות (RadioGroup)"

XML RadioGroup עם RadioButton-ים (רק אחד נבחר בכל פעם):

xml
<RadioGroup android:id="@+id/rg" ... >
    <RadioButton android:id="@+id/rbEng" android:text="English" ... />
    <RadioButton android:id="@+id/rbHeb" android:text="עברית" ... />
</RadioGroup>

JAVA שואלים איזה id נבחר:

java
RadioGroup rg = findViewById(R.id.rg);
int checkedId = rg.getCheckedRadioButtonId();
if (checkedId == R.id.rbEng) {
    tv.setText(et.getText().toString().toUpperCase());
}
ו. "הצג הודעה קופצת (Toast) במקום TextView"

XML אפשר אפילו בלי TextView — רק EditText + Button.

JAVA שורה אחת של Toast:

java
Toast.makeText(this,
        et.getText().toString(),
        Toast.LENGTH_SHORT).show();
ז. "ספור כמה פעמים נלחץ הכפתור"

XML ללא שינוי (כפתור + TextView).

JAVA משתנה ברמת המחלקה (כדי שיישמר בין לחיצות), ומגדילים אותו:

java
public class MainActivity extends AppCompatActivity {
    int count = 0;                              // נשמר בין לחיצות

    @Override
    protected void onCreate(Bundle b) {
        super.onCreate(b);
        setContentView(R.layout.activity_main);

        Button btn = findViewById(R.id.btnUpdate);
        TextView tv = findViewById(R.id.tvOutput);
        btn.setOnClickListener(v -> {
            count++;
            tv.setText("נלחץ " + count + " פעמים");
        });
    }
}
🎯
המסקנה בכל הווריאציות — השלד זהה (onCreate → setContentView → findViewById → setOnClickListener). מה שמשתנה זה רק אילו רכיבים מציירים ב-XML ומה כותבים בתוך הלחיצה ב-Java (לרוב שורה־שתיים). תדע את השלד בעל-פה — ותפתור כל וריאציה.
