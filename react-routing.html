React עם כמה מסכים — ניתוב (Router)

עד עכשיו בנינו אפליקציות של מסך אחד — ה-App פשוט החזיר את <Home/>. עכשיו נראה איך עוברים לכמה מסכים: ה-App הופך ל"שוטר תנועה" שמחליט איזה URL מציג איזה מסך. נלמד את התיאוריה לעומק, נראה את שתי דרכי הניווט, ואז נפתור תרגיל מלא (Plant Shop) קובץ-קובץ.

🧭
קונטקסט — מה משתנה כשעוברים ממסך יחיד לכמה כמעט כלום! כל מסך נשאר קומפוננטה רגילה בדיוק כמו Home שאתה כבר מכיר (state, fetch, JSX). מה שמתווסף זה רק שכבה דקה אחת — Router — שתפקידה היחיד הוא: "לפי כתובת ה-URL, איזה מסך להציג". מ-2 מסכים ל-10 זה בדיוק אותו דפוס — עוד שורת <Route> לכל מסך.

1️⃣ מה זה Router? — התיאוריה

בדפדפן, כל אתר "רגיל" עובד כך: לוחצים על קישור → הדפדפן שולח בקשה חדשה לשרת → השרת מחזיר עמוד HTML חדש → הדף נטען מחדש (הבהוב לבן). זה איטי, וכל מעבר מסך = round-trip לשרת.

React הוא Single Page Application (SPA) — אפליקציה של "דף אחד". כל הקוד (כל המסכים) נטען פעם אחת, ומאז המעבר בין מסכים קורה בצד הלקוח בלבד, בלי טעינה מחדש ובלי לפנות לשרת. מי שמנהל את זה הוא ה-Router (הספרייה react-router-dom).

🧭
מה Router עושה בפועל ה-Router "מאזין" לכתובת ה-URL בשורת הכתובת. כשהיא משתנה (לחיצה על קישור, או ניווט מהקוד) — הוא לא פונה לשרת; במקום זה הוא בודק בטבלת הניתוב שלך "איזו קומפוננטה משויכת לכתובת הזו?", ומחליף את המסך המוצג בלי לרענן את הדף. ה-URL בשורת הכתובת מתעדכן (כדי שאפשר יהיה לסמן/לרענן), אבל הכל קורה מקומית.
🧩
3 חלקי הניתוב
  • הגדרת המסלולים (Routes) — טבלה שאומרת "כתובת X → קומפוננטה Y". זה ה-App.
  • המסכים — קומפוננטות רגילות (Home, About...). אחת לכל מסלול.
  • הניווט — איך עוברים: קישור (<Link>) או קוד (history.push).

2️⃣ מה משתנה ב-App — נכנס Router

עד עכשיו (מסך יחיד) ה-App שלך פשוט החזיר <Home/>. עכשיו ה-App הופך ל"שוטר תנועה": הוא מגדיר איזה URL מציג איזה מסך. זה כל התפקיד שלו:

jsx · App.jsx
function App() {
  const { BrowserRouter, Switch, Route } = ReactRouterDOM;
  return (
    <BrowserRouter>
      <Switch>
        <Route exact path="/"      component={Home} />
        <Route       path="/about" component={About} />
      </Switch>
    </BrowserRouter>
  );
}
window.App = App;

נפרק שורה-שורה:

חלקמה הוא עושה
BrowserRouterעוטף את הכל ומפעיל את מנגנון הניתוב. בלעדיו אין ניתוב — חייב להיות ברמה העליונה.
Switchמסתכל על ה-URL ומציג את ה-Route הראשון שמתאים — ורק אותו (לא כמה במקביל).
Route path="/x" component={Screen}"אם ה-URL הוא /x → הצג את הקומפוננטה Screen".
exact (על "/")בלי זה, / היה מתאים לכל נתיב (כי כל נתיב מתחיל ב-/). exact = "התאמה מדויקת בלבד".
💡
App כבר לא מחזיק UI אמיתי הוא רק מפת הניתוב. אין בו טפסים, רשימות או state — כל אלה עברו למסכים עצמם. כל מסך הוא קומפוננטה נפרדת, וה-Router בוחר מי מהם מוצג לפי הכתובת.
🔤
למה const { BrowserRouter, Switch, Route } = ReactRouterDOM; כשטוענים את react-router-dom מ-CDN דרך <script> (כמו ב-Babel-in-browser שהקורס משתמש בו), הוא יושב על המשתנה הגלובלי ReactRouterDOM. ה-destructuring רק שולף ממנו את הרכיבים שאנחנו צריכים. (בפרויקט עם build אמיתי היינו כותבים import { BrowserRouter } from 'react-router-dom'.)

3️⃣ ניווט בין מסכים — Link מול useHistory

מסך נוסף הוא בדיוק קומפוננטה רגילה כמו שאתה כבר יודע. מה שחדש זה רק איך מנווטים אליו וממנו. יש שתי דרכים, ולכל אחת מקום מתאים:

3.1 <Link> — ניווט הצהרתי (כמו <a>)

קישור שהמשתמש לוחץ עליו. בדרך כלל בתוך תפריט/כפתור ניווט:

jsx · Link
function Home() {
  const { Link } = ReactRouterDOM;
  // ... כל הלוגיקה של המסך כמו קודם ...
  return (
    <div>
      <h1>Plant Shop</h1>
      <Link to="/about">About</Link>     {/* לחיצה → עוברים ל-/about */}
      {/* ... הטופס והרשימה ... */}
    </div>
  );
}
window.Home = Home;
🔗
למה <Link> ולא <a>? <a href> רגיל היה גורם לטעינה מחדש של כל הדף (round-trip לשרת) — בדיוק מה שאנחנו מנסים להימנע ממנו. <Link to> מבצע את המעבר בצד הלקוח, בלי רענון. מבחוץ הוא נראה כמו קישור רגיל.

3.2 useHistory() — ניווט תכנותי (מתוך הקוד)

כשרוצים לנווט מתוך פונקציה — למשל אחרי פעולה מוצלחת (POST הסתיים → תחזור הביתה):

jsx · useHistory
function About() {
  const { useHistory } = ReactRouterDOM;
  const history = useHistory();

  return (
    <div>
      <h1>About Us</h1>
      <p>Plant Shop — the best plants in town 🌱</p>
      <button onClick={() => history.push('/')}>Back to Home</button>
    </div>
  );
}
window.About = About;
🔤
history.push('/') "עבור למסך הזה — מתוך הקוד". משתמשים בזה כשהמעבר הוא תוצאה של פעולה ולא של לחיצה ישירה על קישור (למשל: בסיום שמירה, אחרי התחברות מוצלחת). זה השקול ל-navigation.navigate('Home') שתראה ב-React Native.

3.3 אל תשכח — <script> לכל מסך חדש ב-index.html

בקורס טוענים את הקומפוננטות דרך תגי <script> ב-index.html. כל מסך חדש = עוד שורה:

html · index.html
<script type="text/babel" src="/components/About.jsx"></script>
⚠️
סדר הטעינה חשוב קומפוננטה חייבת להיטען לפני מי שמשתמש בה. לכן App.jsx (שמשתמש ב-Home, About...) נטען אחרון, וה-mount (ReactDOM...render(<App/>)) אחרי כולם.
🧠
סיכום ההכללה
  • App = מפת הניתוב (Router + Routes). מוסיף מסך = מוסיף שורת <Route>.
  • כל מסך = קומפוננטה רגילה. מנווטים עם <Link to="..."> (הצהרתי) או history.push('...') (תכנותי).
  • מ-2 מסכים ל-10 — בדיוק אותו דבר: עוד <Route> לכל מסך.

4️⃣ תרגיל מלא — Plant Shop

4.3 Home.jsx — רשימה + קישור למסך ההוספה

jsx · Home.jsx
function Home() {
  const { Link } = ReactRouterDOM;
  const [plants, setPlants] = React.useState([]);

  async function loadPlants() {
    const res = await fetch('/api/plants');
    const data = await res.json();
    setPlants(data);
  }

  React.useEffect(() => { loadPlants(); }, []);

  async function handleDelete(id) {
    await fetch(`/api/plants/${id}`, { method: 'DELETE' });
    loadPlants();
  }

  return (
    <div>
      <h1>Plant Shop</h1>
      <Link to="/add">+ Add Plant</Link>

      {plants.map(plant => (
        <PlantCard key={plant._id} plant={plant} onDelete={handleDelete} />
      ))}
    </div>
  );
}
window.Home = Home;
🔁
המסך הזה כבר מוכר לך זה בדיוק ה-Home ממסך יחיד (state + useEffect + fetch + map + מחיקה דרך prop) — התוספת היחידה היא שורת ה-<Link to="/add">. שים לב שהטופס כבר לא כאן — הוא עבר למסך נפרד.

4.4 AddPlant.jsx — טופס, ואחרי POST → חזרה הביתה

jsx · AddPlant.jsx
function AddPlant() {
  const { useHistory } = ReactRouterDOM;
  const history = useHistory();

  const [name, setName]       = React.useState('');
  const [species, setSpecies] = React.useState('');
  const [price, setPrice]     = React.useState('');

  async function handleAdd() {
    await fetch('/api/plants', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ name, species, price: Number(price) })
    });
    history.push('/');     // ← חזרה אוטומטית למסך הבית אחרי ההוספה
  }

  return (
    <div>
      <h1>Add Plant</h1>
      <input placeholder="Name"    value={name}    onChange={e => setName(e.target.value)} />
      <input placeholder="Species" value={species} onChange={e => setSpecies(e.target.value)} />
      <input placeholder="Price"   value={price}   onChange={e => setPrice(e.target.value)} />
      <button onClick={handleAdd}>Save</button>
    </div>
  );
}
window.AddPlant = AddPlant;
🎯
הנקודה החשובה כאן אחרי ה-POST אנחנו לא צריכים לרענן רשימה (אין כאן רשימה) — אנחנו פשוט חוזרים הביתה עם history.push('/'). וכשנגיע ל-Home, ה-useEffect שלו ירוץ ויטען את הרשימה מחדש — כך הצמח החדש יופיע. זה הניווט התכנותי בפעולה.

5️⃣ 3 הדברים לראות בעיניים

👀
מה בדיוק קרה כאן
  1. App הפך למפת ניתוב בלבד — שתי שורות <Route>. הטופס שהיה בתוך Home עבר למסך נפרד (AddPlant).
  2. שתי דרכי הניווט בפעולה — <Link to="/add"> (לחיצה הצהרתית) ב-Home, ו-history.push('/') (תכנותי, אחרי ה-POST) ב-AddPlant.
  3. כל מסך מנהל את ה-state שלו — Home טוען את הרשימה, AddPlant מחזיק את שדות הטופס. אין שיתוף — כל מסך עצמאי.
Home (/)רשימה + Delete + קישור "Add Plant"
←
Link to="/add"הדפדפן עובר ל-/add (בלי רענון)
←
AddPlant (/add)ממלאים טופס → Save → POST
←
history.push('/')חזרה ל-Home → useEffect טוען מחדש
🔀
שיתוף נתון בין מסכים? במבחן ברמה הזו — לא צריך. כל מסך עצמאי, וה"מקור האמת" המשותף הוא ה-שרת (כל מסך עושה fetch משלו). אם בכל זאת נדרש לשתף state ישירות בין מסכים בלי לרדת לשרת — זה היה עובר דרך Context (או הרמת state לקומפוננטת אב משותפת), אבל זה מעבר לנדרש כאן.

6️⃣ העשרה — העברת פרמטר בין מסכים (/plants/:id)

מה אם נרצה מסך פרטים — לחיצה על צמח פותחת עמוד עם המידע המלא שלו? זה הדפוס האחרון שמשלים כיסוי מלא. הרעיון: שמים את ה-id בתוך ה-URL, והמסך שולף אותו ועושה fetch לפי ה-id.

6.1 מוסיפים מסלול עם פרמטר ב-App

jsx · App.jsx
<Route exact path="/"           component={Home} />
<Route       path="/add"        component={AddPlant} />
<Route       path="/plants/:id" component={PlantDetails} />   {/* :id = פרמטר */}
🔤
:id — חלק דינמי בנתיב הנקודתיים אומרות "כאן יבוא ערך משתנה". /plants/42, /plants/7 — כולם מתאימים ל-/plants/:id, וה-id יהיה "42" / "7" בהתאמה.

6.2 מנווטים אל המסך עם ה-id (מתוך הכרטיס)

jsx · PlantCard.jsx
function PlantCard({ plant, onDelete }) {
  const { Link } = ReactRouterDOM;
  return (
    <div className="card">
      <Link to={`/plants/${plant._id}`}>{plant.name}</Link>   {/* שם הצמח → קישור לפרטים */}
      <p>{plant.species} · ${plant.price}</p>
      <button onClick={() => onDelete(plant._id)}>Delete</button>
    </div>
  );
}
window.PlantCard = PlantCard;

6.3 המסך החדש שולף את ה-id עם useParams

jsx · PlantDetails.jsx
function PlantDetails() {
  const { useParams } = ReactRouterDOM;
  const { id } = useParams();                 // שולף את ה-:id מה-URL
  const [plant, setPlant] = React.useState(null);

  React.useEffect(() => {
    async function load() {
      const res = await fetch(`/api/plants/${id}`);   // fetch לפי ה-id
      setPlant(await res.json());
    }
    load();
  }, [id]);                                   // ירוץ מחדש אם ה-id משתנה

  if (!plant) return <p>Loading...</p>;        // עד שהנתון מגיע

  return (
    <div>
      <h1>{plant.name}</h1>
      <p>{plant.species} · ${plant.price}</p>
    </div>
  );
}
window.PlantDetails = PlantDetails;
🎯
הדפוס המלא של "מסך פרטים" (1) מסלול עם :id ב-App · (2) <Link to={`/plants/${id}`}> שמנווט אליו · (3) useParams() ששולף את ה-id, ו-fetch לפי ה-id ב-useEffect. שים לב ל-if (!plant) return ... — בזמן שה-fetch רץ, plant עדיין null, אז מציגים "Loading".
📌
למה [id] ב-useEffect ולא []? כדי שאם המשתמש עובר מ-/plants/1 ל-/plants/2 בלי לצאת מהמסך — ה-useEffect ירוץ שוב ויביא את הצמח החדש. [] היה רץ פעם אחת בלבד ונשאר תקוע על הצמח הראשון.
🎯
המשפט אחד לזכור App = מפת ניתוב (BrowserRouter + Switch + Route-ים). כל מסך = קומפוננטה רגילה ועצמאית. מנווטים עם <Link> (לחיצה) או history.push (קוד). פרמטר בנתיב = :id + useParams(). וזהו — מ-2 מסכים ל-N זה אותו דפוס בדיוק.
