Android — Intents, Lists ובקשות HTTP

ממשיכים בתרגול Android מעשי: מעבר בין מסכים והעברת נתונים עם Intents, הצגת רשימות עם ListView + Adapter, ולבסוף בקשות HTTP לשרת עם ספריית OkHttp (כולל עבודה נכונה עם threads).

📑 תוכן העניינים

  2. Intents — מעבר והעברת נתונים

1. אינטראקציה עם Views

java · קריאה מ-EditText וכתיבה ל-TextView
Button btn = findViewById(R.id.button);
btn.setOnClickListener(v -> {
    v.setEnabled(false);              // משבית את הכפתור
    ((Button) v).setText("clicked");
});

Button btnSetter = findViewById(R.id.set_btn);
btnSetter.setOnClickListener(v -> {
    TextView label = findViewById(R.id.textView_set);
    EditText input = findViewById(R.id.editTextText);
    String text = input.getText().toString();   // קריאת קלט
    label.setText(text);                          // הצגתו
});
🔤
פעולות נפוצות על Views input.getText().toString() — קריאת טקסט מ-EditText. label.setText(...) — כתיבה ל-TextView. v.setEnabled(false) — השבתת רכיב. ב-listener, v הוא ה-View שנלחץ (אפשר לעשות לו cast).

2. Intents — מעבר והעברת נתונים

🧭
קונטקסט כדי לעבור למסך אחר ולהעביר אליו נתונים, יוצרים Intent, מצרפים נתון עם putExtra, וקוראים ל-startActivity. במסך היעד שולפים את הנתון מה-Intent.
java · המסך השולח
Button move = findViewById(R.id.button3);
move.setOnClickListener(v -> {
    String text = ((EditText) findViewById(R.id.editTextText)).getText().toString();

    Intent i = new Intent(this, MainActivity2.class);  // יעד: MainActivity2
    i.putExtra("message", text);                       // מצרף נתון
    startActivity(i);                                  // עובר למסך
});
java · המסך המקבל — MainActivity2
public class MainActivity2 extends AppCompatActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main2);

        Intent i = getIntent();                          // ה-Intent שהפעיל את המסך
        String message = i.getStringExtra("message");    // שולף את הנתון
        TextView tv = findViewById(R.id.textView2);
        tv.setText(message);
    }
}
🔤
putExtra / getStringExtra new Intent(this, Target.class) — explicit intent למסך מסוים. putExtra("key", value) מצרף נתון תחת מפתח; במסך היעד getIntent().getStringExtra("key") שולף אותו (יש גם getIntExtra וכו'). זה ה-mechanism מהרצאה 11 — בדיוק כמו props בין מסכים.

3. ListView + Adapter + ניווט

🧭
קונטקסט מציגים רשימה עם ListView + ArrayAdapter (תבנית ה-Adapter מהרצאה 11), ובלחיצה על פריט עוברים למסך אחר עם ה-id של הפריט.
java · ListView + ArrayAdapter
ListView listView = findViewById(R.id.list_view);

ArrayList<String> items = new ArrayList<>();
for (int i = 0; i < 10; i++) {
    items.add(String.format("Item %d", i + 1));
}

ArrayAdapter<String> adapter = new ArrayAdapter<String>(
    getApplicationContext(),
    android.R.layout.simple_list_item_1,   // layout מובנה לכל שורה
    items);
listView.setAdapter(adapter);

// לחיצה על פריט → מעבר למסך אחר עם ה-id:
listView.setOnItemClickListener((parent, view, position, id) -> {
    Intent intent = new Intent(getApplicationContext(), OtherActivity.class);
    intent.putExtra("itemID", position + 1);
    startActivity(intent);
});
🔤
ה-Adapter ArrayAdapter מחבר מערך נתונים ל-ListView, עם layout-שורה מובנה (simple_list_item_1). setOnItemClickListener מקבל את position של הפריט שנלחץ — ומשם יוצרים Intent עם ה-id ועוברים למסך הפרטים.

4. בקשות HTTP עם OkHttp

🧭
קונטקסט כדי לדבר עם שרת (כמו ה-API מתרגול 9), משתמשים בספריית OkHttp. הבקשה אסינכרונית — רצה על thread נפרד (זוכרים מהרצאה 11: אסור לחסום את ה-UI thread!), והתשובה מגיעה ב-callback.
java · HttpClient.java
import okhttp3.Callback;
import okhttp3.OkHttpClient;
import okhttp3.Request;

public class HttpClient {
    private static final String BASE_URL = "https://reqbin.com/echo/get/json";
    private final OkHttpClient client = new OkHttpClient();

    public void fetchUsers(Callback callback) {
        Request request = new Request.Builder().url(BASE_URL).build();
        client.newCall(request).enqueue(callback);   // אסינכרוני!
    }
}
java · MainActivity — שימוש + טיפול בתשובה
private void fetchData() {
    new HttpClient().fetchUsers(new Callback() {
        @Override
        public void onFailure(Call call, IOException e) {
            // ⚠️ ה-callback רץ על thread אחר — חוזרים ל-UI thread לעדכון המסך:
            runOnUiThread(() ->
                Toast.makeText(MainActivity.this, "Failed to fetch data", Toast.LENGTH_SHORT).show());
        }

        @Override
        public void onResponse(Call call, Response response) throws IOException {
            String json = response.body().string();
            // המרת JSON לאובייקטים עם Gson:
            // Gson gson = new Gson();
            // List<User> users = gson.fromJson(json, ...);
            runOnUiThread(() -> tv.setText(json));   // עדכון UI ב-UI thread
        }
    });
}
🧵
‎runOnUiThread‎ — קריטי! enqueue מריץ את הבקשה על thread רשת נפרד, ולכן ה-callbacks (onResponse/onFailure) לא רצים על ה-UI thread. אסור לגעת ב-UI מ-thread אחר (הרצאה 11!) — לכן עוטפים כל עדכון מסך ב-runOnUiThread(() -> ...).
🔤
OkHttp ו-Gson new Request.Builder().url(...).build() בונה בקשה (Builder pattern). client.newCall(request).enqueue(callback) שולח אסינכרונית. response.body().string() מחזיר את ה-JSON כמחרוזת, ו-Gson (gson.fromJson(...)) ממיר אותו לאובייקטי Java. (זה בדיוק מה ש-Retrofit עשה אוטומטית בהרצאה 12.)

★ סיכום

🎯
מה לקחנו
  • Views: getText()/setText()/setEnabled() דרך findViewById.
  • Intents: new Intent(this, Target.class) + putExtra + startActivity; במקבל getIntent().getStringExtra.
  • ListView: ArrayAdapter + setOnItemClickListener (עם position) → ניווט עם id.
  • HTTP: OkHttp enqueue (אסינכרוני), callback עם onResponse/onFailure, תמיד runOnUiThread לעדכון UI, ו-Gson להמרת JSON.
