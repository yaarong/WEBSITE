1. טיפוסים, DateTime ו-StringBuilder

csharp · טיפוסי יסוד
string stringVar = "Hello World!!";
int    intVar    = 100;
float  floatVar  = 10.2f;
char   charVar   = 'A';
bool   boolVar   = true;
Console.WriteLine("intVar: {0}", intVar);   // פלט עם placeholder {0}
csharp · DateTime ו-TimeSpan
DateTime dt  = new DateTime();                       // 01/01/0001 00:00:00
DateTime dt3 = new DateTime(2015, 12, 31);           // שנה, חודש, יום
TimeSpan time = new TimeSpan(10, 5, 25, 50);

Console.WriteLine(dt3 + time);    // חיבור תאריך + משך זמן
Console.WriteLine(dt3 > dt);      // אופרטורי השוואה עובדים על תאריכים

// המרת מחרוזת לתאריך (בטוח, עם out parameter):
var isValid = DateTime.TryParse("5/12/2020", out DateTime parsed);
🔤
StringBuilder ו-TryParse
  • מחרוזות ב-C# הן immutable — לכן לשינויים רבים משתמשים ב-StringBuilder (Append, AppendLine, Replace) שיעיל יותר.
  • DateTime.TryParse(str, out result) — המרה בטוחה: מחזירה bool (הצליח?) ושמה את התוצאה ב-out parameter (פרמטר פלט).

2. var, dynamic ו-Anonymous types

csharp · var (טיפוס מוסק בקומפילציה)
var i   = 10;             // int
var str = "Hello";        // string
var arr = new[] { 1, 2, 3 };   // int[]
// הטיפוס נקבע בזמן קומפילציה ולא ניתן לשינוי (כמו auto ב-C++!)
csharp · dynamic ו-anonymous
// dynamic — הטיפוס נקבע בזמן ריצה ויכול להשתנות:
dynamic d = 100;          d = "Hello";   d = true;   // הכל מותר!

// Anonymous type — אובייקט בלי מחלקה מוגדרת:
var student = new { Id = 1, FirstName = "James", LastName = "Bond" };
Console.WriteLine(student.FirstName);   // James
// student.Id = 2;   ❌ Error — anonymous types הם immutable
🔤
var מול dynamic var — סוכר תחבירי: הקומפיילר מסיק את הטיפוס בזמן קומפילציה, והוא קבוע (בדיוק כמו auto ב-C++ מתרגול 1). dynamic — הטיפוס נקבע בזמן ריצה וניתן לשנותו (כמו JavaScript). Anonymous type — אובייקט מהיר בלי הגדרת מחלקה, immutable.

3. מחלקות ו-Properties

🧭
קונטקסט C# מציעה properties — תחביר נקי ל-getters/setters שמסתיר שדה פרטי מאחורי ממשק נוח (במקום מתודות getX()/setX() כמו ב-Java).
csharp · properties + inheritance
class Person {
    protected int id;
    public int PersonId {
        get { return id; }       // getter
        set { id = value; }      // setter (value = הערך שמושם)
    }
}

class Student : Person {          // ירושה
    public int StudentId {
        get { return id; }
        set { if (value > 0) id = value; }   // לוגיקה ב-setter (ולידציה)
    }
}

// Auto-property — כש-getter/setter פשוטים, C# מייצר את השדה לבד:
class Student2 : Person {
    public int StudentId { get; set; }
}
🔤
Properties get מחזיר ערך, set מקבל אותו דרך מילת המפתח value. אפשר לשים בהם לוגיקה (כמו ולידציה ב-StudentId). אם הם טריוויאליים — auto-property ({ get; set; }) והקומפיילר מייצר את השדה. שימוש: s.StudentId = 5; (נראה כמו שדה, מתנהג כמתודה).

4. Delegates

🧭
קונטקסט Delegate = "מצביע לפונקציה" מטיפוס-בטוח — משתנה שמחזיק התייחסות לפונקציה (או לכמה!). זה הבסיס לאירועים ול-callbacks ב-C#.
csharp · delegate (multicast)
public delegate void MyDelegate(string message);   // הגדרת סוג ה-delegate

void Method1(string m) { Console.WriteLine("Method1: " + m); }
void Method2(string m) { Console.WriteLine("Method2: " + m); }

MyDelegate d = Method1;
d("Hello!");                  // קורא ל-Method1

d += Method2;                 // multicast — מוסיף עוד פונקציה
d("Hello both!");             // קורא לשתיהן!

d -= Method1;                 // מסיר
d("Only Method2");            // קורא רק ל-Method2
🔤
Multicast delegate delegate void MyDelegate(string) מגדיר סוג שמחזיק פונקציות עם החתימה הזו. אפשר לצרף כמה פונקציות עם += (multicast) ולהסיר עם -= — קריאה אחת מפעילה את כולן לפי הסדר. (זה המנגנון מאחורי events — בדיוק ה-Listener pattern, ברמת השפה.)

5. LINQ — Language Integrated Query

🧭
קונטקסט LINQ מאפשר לשאול אוספים (וגם מסדי נתונים, XML...) בתחביר דמוי-SQL בתוך הקוד. שתי צורות: query syntax (דמוי-SQL) ו-method syntax (שרשור מתודות עם lambdas).
csharp · LINQ — שתי הצורות
List<int> numbers = new List<int>() { 6, 5, 3, 4, 9, 1, 7, 10 };

// Query syntax (דמוי-SQL):
var evens = from num in numbers
            where num % 2 == 0
            select num;

// Method syntax (lambda):
var evens2 = numbers.Where(num => num % 2 == 0);

foreach (var n in evens) Console.WriteLine(n);   // 6, 4, 10

// עובד גם על מחרוזות:
var tutorials = from s in stringList
                where s.Contains("Tutorials")
                select s;
🔤
Query מול Method syntax שתיהן שקולות. Query: from ... where ... select ... (קריא כמו SQL). Method: collection.Where(x => ...) עם lambda (מזכיר את filter/map מ-JavaScript בתרגול 4, ואת אלגוריתמי ה-STL מתרגול 3). מתודות נפוצות: Where, Select, OrderBy, Contains.
