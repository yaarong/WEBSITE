1. Hello World — Activity ו-Layout

java · MainActivity.java
public class MainActivity extends AppCompatActivity {
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);   // טוען את ה-layout
    }
}
xml · activity_main.xml
<androidx.constraintlayout.widget.ConstraintLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <TextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello World!"
        app:layout_constraintBottom_toBottomOf="parent"
        app:layout_constraintTop_toTopOf="parent"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintEnd_toEndOf="parent" />

</androidx.constraintlayout.widget.ConstraintLayout>
🔤
תזכורת מהרצאה 11 onCreate טוען את ה-layout עם setContentView(R.layout.activity_main). ב-ConstraintLayout, ארבעת ה-constraint ל-parent "נועלים" את ה-TextView במרכז. match_parent = כל המסך, wrap_content = לפי התוכן.

2. ה-layout של הטיימר

🧭
קונטקסט טיימר עם תצוגת זמן (TextView) ושני כפתורים: start/pause ו-reset. הפעם RelativeLayout (מסדר רכיבים יחסית זה לזה), ולכל רכיב id כדי לגשת אליו מ-Java.
xml · activity_main.xml (טיימר)
<RelativeLayout ...>
    <TextView
        android:id="@+id/countdown_text"
        android:layout_centerInParent="true"
        android:textSize="50sp"
        tools:text="10:00" />

    <Button
        android:id="@+id/countdown_button"
        android:text="start"
        android:layout_below="@id/countdown_text"
        android:layout_centerHorizontal="true" />

    <Button
        android:id="@+id/reset_button"
        android:text="reset"
        android:layout_below="@id/countdown_text"
        android:layout_toEndOf="@id/countdown_button"
        android:visibility="invisible" />   <!-- מוסתר בהתחלה -->
</RelativeLayout>
🔤
RelativeLayout ו-visibility layout_below/layout_toEndOf ממקמים רכיב יחסית לרכיב אחר (לפי id). android:visibility="invisible" מסתיר את כפתור ה-reset עד שצריך אותו. (tools:text מוצג רק בעורך, לא בריצה.)

3. חיבור ל-לוגיקה + אירועים

java · onCreate — findViewById + listeners
public class MainActivity extends Activity {
    private static final long TOTAL_TIME = 10_000;   // 10 שניות (ms)

    private TextView countDownText;
    private Button countDownButton, resetButton;
    private CountDownTimer timer;
    private long timeLeftInMilliSeconds;
    private boolean timerRunning = false;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        // מאתרים את ה-Views לפי ה-id-ים מה-XML:
        countDownText   = findViewById(R.id.countdown_text);
        countDownButton = findViewById(R.id.countdown_button);
        resetButton     = findViewById(R.id.reset_button);

        // רושמים מאזיני לחיצה (Listener pattern):
        countDownButton.setOnClickListener(view -> startStop());
        resetButton.setOnClickListener(view -> resetTimer());

        resetTimer();
    }

    public void startStop() {
        if (timerRunning) pauseTimer();
        else              startTimer();
    }
🔤
הגשר XML ↔ Java findViewById(R.id.countdown_text) מאתר את ה-View לפי ה-id שהגדרנו ב-XML. setOnClickListener(view -> ...) רושם מאזין לחיצה (בתחביר lambda) — בדיוק ה-Listener pattern מהרצאה 11. שומרים את ה-Views כשדות כדי לגשת אליהם אחר כך.

4. CountDownTimer

🧭
קונטקסט CountDownTimer הוא כלי מובנה ב-Android לספירה לאחור. מגדירים אותו עם זמן כולל ותדירות "טיקים", ומממשים שתי מתודות: onTick (בכל טיק) ו-onFinish (בסיום).
java · startTimer
public void startTimer() {
    timer = new CountDownTimer(timeLeftInMilliSeconds, 100) {  // (זמן כולל, טיק כל 100ms)
        @Override
        public void onTick(long l) {
            timeLeftInMilliSeconds = l;
            updateTimer();                  // מעדכן את התצוגה בכל טיק
        }
        @Override
        public void onFinish() {
            timerRunning = false;
            countDownButton.setVisibility(View.INVISIBLE);
            resetButton.setVisibility(View.VISIBLE);
            Toast.makeText(getApplicationContext(), "Time's Up!", Toast.LENGTH_LONG).show();
        }
    };
    timer.start();

    countDownButton.setText("pause");
    resetButton.setVisibility(View.INVISIBLE);
    timerRunning = true;
}

public void pauseTimer() {
    timer.cancel();
    countDownButton.setText("start");
    resetButton.setVisibility(View.VISIBLE);
    timerRunning = false;
}
🔤
CountDownTimer הבנאי מקבל את הזמן הכולל ואת מרווח הטיקים (ms). onTick(l) נקרא בכל טיק עם הזמן שנותר; onFinish() בסיום. timer.start() מתחיל, timer.cancel() עוצר. Toast = הודעה קופצת קצרה.

5. עדכון ה-UI

java · updateTimer + resetTimer
public void resetTimer() {
    timeLeftInMilliSeconds = TOTAL_TIME;
    updateTimer();
    resetButton.setVisibility(View.INVISIBLE);
    countDownButton.setVisibility(View.VISIBLE);
}

public void updateTimer() {
    int minutes = (int) timeLeftInMilliSeconds / 60_000;            // דקות
    int seconds = (int) (timeLeftInMilliSeconds % 60_000) / 1000;   // שניות

    StringBuilder sb = new StringBuilder();
    sb.append(minutes).append(":");
    if (seconds < 10) sb.append(0);     // ריפוד אפס: "1:05" במקום "1:5"
    sb.append(seconds);

    countDownText.setText(sb.toString());   // מעדכן את התצוגה
}
🔤
עדכון תצוגה
  • setText(...) — משנה את הטקסט המוצג ב-TextView/Button.
  • setVisibility(View.VISIBLE / INVISIBLE) — מציג/מסתיר רכיב.
  • החישוב: / 60_000 = דקות (60,000ms בדקה), % 60_000 / 1000 = שניות שנותרו. StringBuilder בונה את מחרוזת התצוגה ("דקות:שניות").

★ סיכום

🎯
מה לקחנו
  • Activity + Layout: onCreate → setContentView; ConstraintLayout ו-RelativeLayout; id לכל רכיב, visibility.
  • חיבור: findViewById(R.id...) + setOnClickListener (Listener pattern).

  • עדכון UI: setText, setVisibility, Toast, ו-StringBuilder לבניית התצוגה.
