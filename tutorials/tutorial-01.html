חלק א' · סינטקס C++

1. std::string

המחליפה של ה-C-string (char *). מחרוזת חכמה שמנהלת את עצמה:

  • תומכת בשינויים — אפשר להוסיף, למחוק ולשנות תווים.
  • יודעת מה הגודל שלה (.size()).
  • לא צריך לדאוג להקצאה/שחרור זיכרון בעבורה.
  • אפשר להמיר ל-C-string עם .c_str() (קבוע), או .data() (ניתן לשינוי, מ-C++17).
cpp · std::string
#include <string>

std::string s = "Hello";
s += ", World!";          // שרשור — לא היה אפשרי עם char*
std::cout << s.size();     // 13
const char* c = s.c_str(); // המרה ל-C-string קבוע

2. Namespaces ו-using

🧭
הבעיה שזה פותר ב-C מרחב השמות מצומצם — אי אפשר שתי פונקציות עם אותו שם. Namespaces ב-C++ מאפשרים לארגן שמות במרחבים נפרדים, כך שאותו שם יכול להופיע בכמה מקומות.
cpp · הגדרה ושימוש
namespace Math {
    int add(int a, int b) { return a + b; }
}
namespace Physics {
    int add(int a, int b) { return a + b + 1; }   // אותו שם, מרחב אחר
}

int x = Math::add(2, 3);      // גישה עם scope resolution operator  ::
int y = Physics::add(2, 3);
🔤
\u200E::\u200E ו-using
  • :: — scope resolution operator. std::cout = ה-cout שבמרחב הספרייה הסטנדרטית std. ::name לבד = המרחב הגלובלי.
  • using std::cout; — מביא חבר ספציפי למרחב הנוכחי כדי לחסוך את הקידומת. כלל אצבע: רק בקבצי .cpp (לא .hpp), ובזהירות — להימנע מ-using namespace std; גורף.
  • אפשר namespaces מקוננים (אחד בתוך השני) ללא הגבלה.

3. קלט/פלט — cin / cout

ב-C השתמשנו ב-printf/scanf (עם <stdio.h>). הם עדיין עובדים, אבל ב-C++ עדיף האובייקטים הגלובליים cin/cout שב-std, עם האופרטורים << (פלט) ו->> (קלט):

cpp · I/O
#include <iostream>

int age;
std::cout << "Enter age: ";
std::cin  >> age;                              // קריאה מהמסך
std::cout << "You are " << age << std::endl;   // שרשור אופרטורים
🔤
\n מול std::endl שניהם יורדים שורה. std::endl בנוסף מרוקן את ה-buffer (flush) — נחשב "יפה" יותר אבל איטי יותר מעט. אפשר לשרשר כמה << ברצף.

4. auto, nullptr, bool

cpp · מילות מפתch
auto num = 25;          // num הוא int — הקומפיילר מסיק את הטיפוס
auto str = "C++";       // str הוא const char*
const auto pi = 3.14;   // אפשר גם const auto, auto* וכו'

int* p = nullptr;       // איפוס מצביע — לא NULL
bool flag = true;       // טיפוס בוליאני: true / false
🔤
מה כל אחד
  • auto — הקומפיילר מסיק את הטיפוס מהערך. נוח, אבל להשתמש בזהירות (פחות מפורש).
  • nullptr — הדרך הנכונה לאפס מצביע. NULL הוא בעצם המספר 0, מה שעלול לבלבל בין מצביעים למספרים; nullptr הוא מסוג מצביע אמיתי, ולא ניתן לאתחל בו מספר.
  • bool — ערכי אמת/שקר (true/false).

5. העמסת פונקציות + ערכי ברירת מחדל

cpp · Overloading & default params
// העמסה — אותו שם, חתימה שונה (סוג/כמות פרמטרים)
int    add(int a, int b)       { return a + b; }
double add(double a, double b) { return a + b; }   // OK

// ערכי ברירת מחדל — חייבים להיות הימניים ביותר
void greet(std::string name, std::string title = "Mr.") {
    std::cout << "Hello " << title << " " << name;
}
greet("Cohen");          // Hello Mr. Cohen
greet("Levi", "Dr.");    // Hello Dr. Levi
⚠️
שני כללים (1) לא ניתן להעמיס לפי ערך ההחזרה — רק לפי הפרמטרים. (2) ערכי ברירת מחדל חייבים להיות הפרמטרים הימניים ביותר (אי אפשר ראשון דיפולטיבי ושני לא).

חלק ב' · מצביעים ורפרנסים

6. איזורי זיכרון

לתוכנית יש ארבעה איזורי זיכרון עיקריים:

איזורמה שמור בוניהול
Textהקוד שרץאוטומטי
Dataמשתנים גלובליים, סטטיים וקבועיםאוטומטי (חי כל חיי התכנית)
Stackמשתנים לוקאליים, פרמטרים, כתובות חזרהאוטומטי (חי עד סוף ה-scope) · מהיר · מוגבל בגודל
Heapהקצאות דינמיותידני (המתכנת מקצה ומשחרר) · גדול · עלול להיכשל
💡
כלל אצבע משתנים קטנים → Stack (מהיר, אוטומטי). משתנים גדולים / בגודל לא-ידוע בזמן ריצה / שצריך שישרדו מעבר ל-scope → Heap (דינמי). הקצאה על ה-Heap עשויה להיכשל — צריך לבדוק.

7. מצביעים — תחביר

מצביע = משתנה שמחזיק כתובת בזיכרון. גודלו תלוי בביטיות (4 בתים ב-32bit, 8 בתים ב-64bit).

cpp · declare, address-of, dereference
int value = 0;
int* ptr = &value;     // & = address-of: ptr מחזיק את כתובת value

int other = *ptr;       // * = dereference: other = 0 (הערך אליו מצביעים)
*ptr = 5;               // משנה את value עצמו → value == 5

// הצהרת כמה מצביעים בשורה — * לכל אחד:
int *p1, *p2, x;        // p1,p2 מצביעים; x הוא int רגיל
🔤
\u200E*\u200E מול \u200E&\u200E &value — מקבל את הכתובת של משתנה. *ptr — ניגש לערך שבכתובת (dereferencing). חייבת להיות התאמת טיפוסים: int* לא יכול להצביע על כתובת של long.
💥
Null ו-Dangling pointers
  • Null pointer — מצביע ל-nullptr (כתובת 0, לא חוקית). *ptr עליו = Undefined Behavior (בד"כ קריסה).
  • Dangling pointer — מצביע לכתובת שהייתה חוקית ושוחררה (משתנה לוקאלי שיצא מ-scope, או זיכרון Heap ששוחרר). שימוש בו = UB. שתי מלכודות נפוצות: החזרת כתובת של משתנה לוקאלי, ושחרור זיכרון בלי לאפס את המצביע ל-nullptr.

8. מצביעים, מערכים ואריתמטיקה

מערך הוא אוסף איברים רציף בזיכרון, ולכן מצביע יכול לשמש כמערך. כתובת איבר מחושבת מהכתובת הבסיסית + אינדקס × גודל-טיפוס:

cpp · pointer arithmetic
int arr[5] = {10, 20, 30, 40, 50};
int* p = arr;            // p מצביע לאיבר הראשון

p[2];                    // 30  — סמנטיקת מערך
*(p + 2);                // 30  — אריתמטיקה (שקול)

// קידום מצביע מתחשב בגודל הטיפוס:
// p+1 על int* מקדם 4 בתים; חיסור מצביעים = מרחק באיברים, לא בבתים
🔤
המרת מצביעים ככלל אין צורך להמיר בין מצביעים. המרה "מסוכנת" נעשית עם reinterpret_cast<NewType>(val), והיא חוקית בעיקר ל-void*, למערכי בתים (char*), או בחזרה לטיפוס המקורי. void* = מצביע "כללי" שיכול להצביע לכל טיפוס, אבל אפשר עליו רק השוואות.

9. const ו-"כלל המעגל"

🎯
כלל המעגל "qualifier תמיד משפיע על מה שמשמאלו — אלא אם הוא השמאלי ביותר, ואז על מה שמימינו."
cpp · const עם מצביעים
const int* p1;        // const שמאלי ביותר → מצביע ל-int קבוע (אי אפשר לשנות את *p1)
int const* p2;        // זהה ל-p1
int* const p3 = &x;   // const משמאל ל-* → המצביע עצמו קבוע (אי אפשר להחליף כתובת)
const int* const p4 = &x;  // גם המצביע וגם הערך קבועים
📌
שתי נקודות מצביע יכול תמיד להוסיף qualifier (כמו const) אבל לא להוריד אותו באופן לא-מפורש. ו-מצביע קבוע חייב להיות מאותחל בהצהרה.

10. רפרנסים (References)

🧭
מה זה Reference = כינוי (alias) למשתנה קיים. אם x הוא reference ל-y, אז הם אותו משתנה בדיוק — אותה כתובת, ושינוי אחד משנה את השני. בניגוד למצביע, רפרנס חייב להיות מאותחל, ולא ניתן ל"קשור מחדש".
cpp · reference
int y = 10;
int& z = y;            // z הוא alias ל-y
z = 99;                // עכשיו y == 99 גם כן

const int& r = y;      // קריאה בלבד — לא ניתן לשנות דרך r

// העברת פרמטר ב-reference — מאפשר לשנות את המקור (כמו swap):
void swap(int& a, int& b) { int t = a; a = b; b = t; }
הבדלמצביערפרנס
בטיחותיכול להיות null / לא מאותחלחייב לאתחל ממשתנה חוקי
קיוםמשתנה עצמאי עם כתובתרק כינוי — לא בהכרח קיים בנפרד
שינוי אחרי הצהרהניתן להצביע למשתנה אחרלא ניתן לקשור מחדש
סמנטיקהPointer (צריך */&)Value (כמו משתנה רגיל)
⚡
const-ref לאובייקטים גדולים שימוש נפוץ מאוד: העברת אובייקט גדול כ-const& — חוסך העתקה (מקבלים את המקור, לא עותק) ובטוח (לא ניתן לשנות). כלל אצבע: אובייקט גדול לקריאה בלבד → const&. ולעולם לא להחזיר רפרנס למשתנה לוקאלי (dangling).

11. Rvalue references ו-move

🧭
lvalue מול rvalue lvalue = משתנה עם שם (אפשר לקחת לו כתובת). rvalue = ערך זמני / ללא שם / לקראת סוף חייו (למשל 2, a + b, ערך מוחזר מפונקציה). C++11 הוסיפה rvalue reference עם &&.
cpp · rvalue reference
int&& m = 42;                         // rvalue reference

void process(std::vector<double>&& vec);  // "אני עומד לקחת את המשאבים שלך"
🔤
למה זה קיים — Move semantics כשזמן חייו של אובייקט נגמר, אפשר להעביר (move) את המשאבים שלו לאובייקט אחר במקום להעתיק אותם — חוסך עבודה. פעולת הזזה מסומנת ב-noexcept (מוסכמה: הזזה לא אמורה להיכשל).
5️⃣
Rule of 5 מחלקה שמכילה אחת מהמתודות הבאות — צריכה (בד"כ) להכיל את כולן:
  1. בנאי העתקה (copy constructor)
  2. בנאי הזזה (move constructor)
  3. אופרטור השמה בהעתקה (copy assignment)
  4. אופרטור השמה בהזזה (move assignment)
  5. משמיד (destructor)

חלק ג' · הקצאות זיכרון

12. new / delete

cpp · C מול C++
// C:
int* i    = (int*) malloc(sizeof(int));
int* iArr = (int*) malloc(sizeof(int) * size);
free(i);
free(iArr);

// C++:
int* i    = new int;
int* iArr = new int[size];
delete i;
delete[] iArr;        // שימו לב ל-[] עבור מערך!
🔤
new/delete מול malloc/free
  • malloc/free — מקצים/משחררים בתים, לא מודעים לטיפוס, מחזירים void*.
  • new/delete — מקצים/משחררים אובייקטים: קוראים לבנאי ולמשמיד, מחשבים את הגודל אוטומטית, יודעים להקצות מערכים, וניתנים ל-override.
💧
Memory Leak ו-Dangling — שתי הסכנות
cpp
int* i = new int(8);
int* j = new int(10);
delete j;
std::cout << *j;   // j מצביע לזיכרון משוחרר → dangling pointer!
// i לעולם לא שוחרר → memory leak!
לכל new חייב להיות delete תואם — אחרת דליפת זיכרון. וגישה אחרי delete = dangling. בדיוק מהסיבות האלה הומצאו smart pointers.

13. Smart Pointers

🧭
קונטקסט לעקוב ידנית אחרי מצביעים זה מקור עיקרי לדליפות זיכרון. Smart pointer = מצביע שיודע לשחרר את עצמו כשנגמר זמן חייו (מ-C++11, בספרייה הסטנדרטית). שלושה סוגים: unique_ptr, shared_ptr, weak_ptr.
cpp · unique_ptr
#include <memory>

// מומלץ ליצור עם make_unique:
std::unique_ptr<int> p = std::make_unique<int>(42);
std::cout << *p;          // 42
// אין צורך ב-delete — ישוחרר אוטומטית כש-p יוצא מ-scope
🔤
שלושת הסוגים
  • unique_ptr — בעלות בלעדית על האובייקט (לא ניתן להעתיק, רק להעביר/move). כמעט בלי overhead — זה ברירת המחדל שנשתמש בה במקום new/delete.
  • shared_ptr — בעלות משותפת: מחזיק reference counter. כל העתקה מעלה ב-1, כל מחיקה מורידה ב-1, וכשמגיע ל-0 — האובייקט נמחק.
  • weak_ptr — "משקיף" על shared_ptr בלי לשנות את ה-counter (לא בעלים). פותר בעיית מעגליות (שני shared_ptr שמצביעים זה על זה לא יגיעו ל-0), וטוב ל-cache.

★ סיכום

🎯
מה לקחנו
  • סינטקס: std::string, namespaces (::/using), cin/cout, auto/nullptr/bool, העמסה + ערכי ברירת מחדל.
  • זיכרון: Data/Text/Stack/Heap; מצביעים (*/&, null/dangling), אריתמטיקה, const (כלל המעגל).
  • רפרנסים: alias למשתנה; const& לאובייקטים גדולים; rvalue refs (&&) ו-move; Rule of 5.
  • הקצאות: new/delete (מול malloc/free), והפתרון לדליפות — smart pointers (unique/shared/weak).
