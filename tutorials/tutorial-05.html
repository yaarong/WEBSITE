1. IPC — תקשורת בין תהליכים

🧭
קונטקסט תהליך (Process) = תוכנית שרצה כרגע על המעבד. לעיתים קרובות נרצה שתהליכים ישתפו מידע — זה IPC (Inter-Process Communication). הפתרון הבסיסי הוא קובץ, אבל לו חסרונות (לא מסונכרן, נשאר עד שמוחקים, פעולות יקרות). פתרון טוב יותר באותה מערכת: Pipe — "צינור" שכותבים אליו מצד אחד וקוראים מהשני.
🔌
Socket — מעבר למחשב אחד לכל פתרונות ה-IPC הרגילים יש חיסרון: הם מוגבלים לאותה מערכת. Socket מאפשר תקשורת בין מחשבים. ברמה גבוהה (TCP), socket מספק "צינור דו-כיווני" שלא מוגבל למחשב המקומי. מאחורי הקלעים sockets לא אמינים (הודעות אובדות/נהרסות/מתחלפות בסדר) — ולכן משתמשים ב-TCP שמספק אמינות.

2. מודל ה-OSI (7 שכבות)

🧭
קונטקסט מודל 7 השכבות מתאר את העברת המידע ברשת באופן סטנדרטי. כל שכבה לא מודעת לשכבות שמעליה, ורואה את שמתחתיה כ"קופסה שחורה". בקורס נעבוד תמיד בשכבת האפליקציה (העליונה).
#שכבהתפקיד · פרוטוקולים
7Applicationמה עושים עם המידע · HTTP, FTP, SSH (כאן אנחנו כותבים)
6Presentationייצוג/הצפנה
5Sessionניהול חיבורים
4Transportאמינות, segmentation, congestion control · TCP, UDP (כאן ה-API)
3Networkכתובות IP וניתוב · IP, ICMP
2Data Linkחלוקת הרשת הפיזית, זיהוי שגיאות · ARP, MAC
1Physicalהעברת ביטים על תווך פיזי (best-effort) · USB, Bluetooth
🔤
השכבות שמעניינות אותנו Transport (4) — כאן ה-API של מערכת ההפעלה (TCP/UDP), והשכבה שמספקת את התכונות שאנחנו מחפשים מ-socket: אמינות, segmentation (הודעות ארוכות לא מתפצלות), ובקרת עומס. Application (7) — כאן אנחנו מגדירים את הפרוטוקול שלנו איך שנרצה.

4. כתובות — IP ו-Port

🔤
כתובת socket = IP + Port
  • IP — כתובת המחשב המארח. IPv4 = 4 בתים, פורמט "נקודות" (למשל 127.0.0.1), כל מספר בטווח 0–255. מוגבל ל-~4 מיליארד כתובות → נוצר IPv6 (16 בתים, hex עם נקודתיים).
  • Port — "כתובת בתוך המחשב", מספר 2 בתים.
טווח Portסוגדוגמה
0–1023well-known (מערכת)HTTP = 80
1024–49151registered (משתמש)MySQL = 3306
>49151dynamic / privateשה-OS נותן בקבלת חיבור

5. זרימת TCP + דוגמת Python

   TCP Server                 TCP Client
   ──────────                 ──────────
   socket()                   socket()
   bind()
   listen()
   accept()  ◀───connect()─── connect()
       │                          │
   write()/read() ◀══════════▶ write()/read()
   close()                    close()
🔤
הזרימה במילים השרת: פותח socket → bind לכתובת ידועה → listen → accept (ממתין לחיבור, ויוצר socket חדש לכל לקוח). הלקוח: פותח socket → connect לכתובת השרת. אחרי החיבור — תקשורת דו-כיוונית בזרם בתים, וסיום ב-close.

ב-Python זה קל כמו פתיחת קובץ. השרת (שולח תמונה ללקוח):

python · TCPserver.py
import socket

def send_image(sock, file_name):
    with open(file_name, 'rb') as file:
        while True:
            data = file.read(1024)
            if not data:
                break
            sock.send(data)

server_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)  # IPv4, TCP
server_socket.bind(('localhost', 12345))
server_socket.listen(1)
print('Server is listening on port 12345')

while True:
    client_socket, addr = server_socket.accept()   # ממתין לחיבור
    print('Accepted connection from', addr)
    send_image(client_socket, 'image.jpg')
    client_socket.close()
python · TCPclient.py
import socket

def receive_image(sock, file_name):
    with open(file_name, 'wb') as file:
        while True:
            data = sock.recv(1024)
            if not data:
                break
            file.write(data)

client_socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
client_socket.connect(('localhost', 12345))
print('Connected to the server')
receive_image(client_socket, 'received_image.jpg')
client_socket.close()
🔤
ה-API של Python socket(AF_INET, SOCK_STREAM) — AF_INET=IPv4, SOCK_STREAM=TCP (ל-UDP: SOCK_DGRAM). bind/listen/accept בשרת; connect בלקוח; send/recv מעבירים bytes (לא מחרוזות). שימו לב לקריאה ב-loop של 1024 בתים — TCP הוא זרם, לא יודעים מראש כמה יגיע.

6. זרימת UDP + דוגמת Python

   UDP Server                 UDP Client
   ──────────                 ──────────
   socket()                   socket()
   bind()
   recvfrom()/sendto() ◀════▶ sendto()/recvfrom()
   close()                    close()
🧭
UDP — חסר חיבור ב-UDP אין connect/accept — כל הודעה עומדת בפני עצמה. משתמשים ב-sendto(data, address) ו-recvfrom(size) (שמחזיר גם את כתובת השולח). השרת זהה ללקוח, רק שהוא bind לכתובת ידועה כדי שלקוחות ימצאו אותו.
python · UDP (קטעי מפתח)
# server:
sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)   # UDP
sock.bind(('localhost', 12345))
data, address = sock.recvfrom(4096)        # מקבל גם את כתובת השולח
sock.sendto(chunk, address)                # שולח לכתובת ספציפית

# client:
sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
sock.settimeout(1)                         # timeout — כי אין הבטחה שתגיע תשובה
sock.sendto(b'This is the message.', ('localhost', 12345))
chunk, server = sock.recvfrom(4096)
💡
למה settimeout ב-UDP ב-UDP הודעות יכולות לאבוד בלי שנדע. settimeout(1) מבטיח שאם לא הגיעה תשובה תוך שנייה — נקבל חריגה ולא ניתקע לנצח. (בקוד המלא יש טיפול ב-TimeoutError שמרכיב את החלקים שכן הגיעו.)

7. Sockets API ב-Unix C

🧭
קונטקסט זרימת ה-TCP זהה ל-Python, אבל ב-C/C++ זה קשה יותר — מערב syscalls ומבני-קרנל. כל קריאה מחזירה מספר שלילי בכישלון.
קריאהתפקיד
socket(domain, type, protocol)יוצר fd חדש (domain=AF_INET, type=SOCK_STREAM/SOCK_DGRAM, protocol=0)
bind(fd, addr, len)כורך כתובת ל-socket
listen(fd, backlog)גודל תור הלקוחות הממתינים
accept(fd, addr, len)מקבל חיבור חדש (TCP)
connect(fd, addr, len)מתחבר לשרת (לקוח)
c · sockaddr_in + יצירת socket
struct sockaddr_in addr;
addr.sin_family = AF_INET;
addr.sin_port = htons(12345);              // ← המרה ל-big-endian!
inet_aton("127.0.0.1", &addr.sin_addr);    // מחרוזת IP → 4 בתים

int fd = socket(AF_INET, SOCK_STREAM, 0);
if (fd < 0) { /* error */ }
bind(fd, (struct sockaddr*)&addr, sizeof(addr));
🔄
Endianness — חובה לזכור! יש שתי דרכים לייצג מספרים בזיכרון: little-endian (הנפוץ במעבדים) ו-big-endian. בתכנות רשתי הסטנדרט הוא big-endian ("network byte order"). לכן חייבים להמיר מספרים (כמו port) לפני שמכניסים אותם למבני התקשורת. הפונקציות:
  • htons / htonl — host→network (short / long)
  • ntohs / ntohl — network→host
(inet_aton ממיר מחרוזת IP טקסטואלית למספר בינארי של 4 בתים.)

★ סיכום

🎯
מה לקחנו
  • IPC: קבצים → Pipe (באותה מערכת) → Socket (בין מחשבים).
  • OSI: 7 שכבות, כל אחת קופסה שחורה לזו שמתחתיה. אנחנו: Transport (TCP/UDP) ו-Application.
  • TCP — אמין, מבוסס-חיבור, זרם. UDP — מהיר, חסר-חיבור, חבילות.
  • כתובת = IP + Port (IPv4=4 בתים, port=2 בתים; טווחי well-known/registered/dynamic).
  • API: שרת socket→bind→listen→accept, לקוח socket→connect, ואז send/recv. ב-Python קל; ב-C צריך לזכור endianness (htons).
