1. יצירת אפליקציה ו-Components

bash · create-react-app
npx create-react-app my-app
cd my-app
npm start                 # שרת פיתוח ב-http://localhost:3000

רכיב ב-React = פונקציה (או מחלקה) שמחזירה JSX. שתי הצורות:

jsx · hello.jsx
import React from 'react';
import ReactDOM from 'react-dom/client';

// Functional Component (מועדף):
function Hello(props) {
    return <h1>Hello React!</h1>;
}

// Class Component (סגנון ישן יותר):
// class Hello extends React.Component {
//     render() { return <h1>Welcome to React!</h1>; }
// }

const root = ReactDOM.createRoot(document.getElementById("root"));
root.render(<Hello />);
🔤
פונקציונלי מול מחלקתי Functional component = פונקציה שמחזירה JSX (קצר, מודרני, ועם Hooks). Class component = מחלקה שיורשת מ-React.Component עם מתודת render(). שניהם עובדים — היום מעדיפים functional.

2. Props ורינדור מותנה

jsx · condition.jsx
function HelloReact(props)  { return <h1>Hello React {props.val}!</h1>; }
function HelloReact2(props) { return <h1>Hello React again {props.val}!</h1>; }

function Hello(props) {
    const flag = props.flag;
    if (flag) {
        return <HelloReact val={props.val} />;     // רינדור מותנה
    }
    return <HelloReact2 val={props.val} />;
}

root.render(<Hello flag={true} val={5} />);
🔤
props ורינדור מותנה
  • props — הנתונים שמעבירים לרכיב כ"מאפיינים" (val={5}, flag={true}). מזריקים ל-JSX עם {props.val}.
  • רינדור מותנה — רכיב יכול להחזיר JSX שונה לפי תנאי (if). כך מציגים תוכן שונה לפי ה-state/props.

3. State — setState

🧭
קונטקסט State = נתונים שהרכיב מנהל ושמשתנים בזמן ריצה. כשה-state משתנה — React מרנדר מחדש. ברכיב מחלקתי משתמשים ב-this.state ו-this.setState:
jsx · counter.jsx
class Counter extends React.Component {
    constructor(props) {
        super(props);
        this.state = { count: 0 };           // אתחול state
    }

    increment = () => {
        this.setState({ count: this.state.count + 1 });   // עדכון state
    };

    render() {
        return (
            <div>
                <p>Count: {this.state.count}</p>
                <button onClick={this.increment}>Increment</button>
            </div>
        );
    }
}
🪤
תמיד setState — לא השמה ישירה משנים state רק דרך this.setState(...) — לעולם לא this.state.count = ... ישירות (React לא ירנדר מחדש). לעדכון שתלוי בערך הקודם, עדיף הצורה הפונקציונלית: this.setState(prev => ({ count: prev.count + 1 })). (ברכיב פונקציונלי זה ה-Hook useState מהרצאה 8.)

4. React.memo — אופטימיזציה

🧭
הבעיה כשרכיב הורה מרנדר מחדש — כל רכיבי הילד מרנדרים מחדש, גם אם ה-props שלהם לא השתנו. זה מבזבז עבודה. React.memo "זוכר" את התוצאה ומדלג על רינדור אם ה-props לא השתנו.
jsx · App.jsx
// ילד רגיל — מרנדר תמיד מחדש כשההורה מרנדר:
const RegularChild = ({ value }) => {
    console.log("RegularChild Rendered");
    return <h3>Regular Child: {value}</h3>;
};

// ילד ממוטב — מרנדר רק כש-value שלו משתנה:
const MemoizedChild = memo(({ value }) => {
    console.log("MemoizedChild Rendered");
    return <h3>Memoized Child: {value}</h3>;
});
⚡
מה רואים אם ההורה מחזיק count ו-otherValue, ולוחצים "Increment Count": RegularChild ירנדר מחדש (גם אם ה-value שלו לא קשור), אבל MemoizedChild (שתלוי ב-otherValue) לא ירנדר — נראה זאת בקונסול. אופטימיזציה ל-UI גדול.

5. React Router

🧭
קונטקסט אפליקציית React היא לרוב SPA (Single Page Application) — דף אחד שמחליף תוכן בלי לטעון מחדש. React Router (react-router-dom) ממפה נתיב URL → רכיב, בלי רענון העמוד.
jsx · main.jsx — הגדרת הנתיבים
import { BrowserRouter, Routes, Route } from "react-router-dom";
import Home from "./pages/Home";
import Navbar from "./pages/Navbar";
import User from "./pages/User";
import NotFound from "./pages/NotFound";

export default function App() {
    return (
        <BrowserRouter>
            <Navbar />
            <Routes>
                <Route index element={<Home />} />
                <Route path="/about"   element={<About />} />
                <Route path="/user/:username" element={<User />} />
                <Route path="*" element={<NotFound />} />  {/* 404 */}
            </Routes>
        </BrowserRouter>
    );
}
jsx · Navbar.jsx + User.jsx
import { Link } from "react-router-dom";
const Navbar = () => (
    <nav>
        <Link to="/">Home</Link>
        <Link to="/about">About</Link>
        <Link to="/user/john">User</Link>
    </nav>
);

// נתיב דינמי — קוראים את הפרמטר עם useParams:
import { useParams } from "react-router-dom";
const User = () => {
    const { username } = useParams();
    return <p>Welcome, {username}!</p>;
};
🔤
רכיבי ה-Router
  • <BrowserRouter> — עוטף את האפליקציה ומפעיל את הניתוב.
  • <Routes> + <Route path element> — ממפה נתיב לרכיב. index = ברירת מחדל; path="*" = כל השאר (404).
  • <Link to="..."> — קישור פנימי בלי רענון (במקום <a> שטוען מחדש).
  • :username = נתיב דינמי; קוראים אותו עם useParams(). (בדיוק כמו req.params בהרצאה 7, וה-[id]+useLocalSearchParams מהרצאה 10!)
🎮
בונוס — משחק XO (tic-tac-toe) התרגול כולל גם מימוש מלא של משחק איקס-עיגול ב-React — דוגמה מצוינת שמשלבת state (לוח המשחק), רינדור רשימה (התאים), ואירועים (לחיצה על תא). שווה לעבור עליו ככלי לתרגול כל מה שלמדנו.

★ סיכום

🎯
מה לקחנו
  • Components: פונקציונליים (מועדף) מול מחלקתיים; כולם מחזירים JSX.
  • Props + רינדור מותנה (if שמחזיר JSX שונה).

  • React.memo — מדלג על רינדור ילד אם ה-props שלו לא השתנו.
