HTML ו-JavaScript בדפדפן — DOM, אירועים וטפסים

מחברים בין HTML ל-JavaScript בצד הלקוח. נלמד דרך דוגמאות אינטראקטיביות: מבנה HTML, אירועים (onclick), מניפולציה של ה-DOM, JS חיצוני, עבודה עם טפסים וקלט, ו-ולידציה — ובניית "שעון דפדפן" ו"רשימת קניות".

🎯
קונטקסט בתרגול 4 למדנו JavaScript "טהור". כאן ה-JS פוגש את הדף: הוא קורא ומשנה אלמנטים ב-HTML דרך ה-DOM (Document Object Model) — עץ האובייקטים שמייצג את העמוד.

1. מבנה HTML בסיסי

html · hello_html.html
<!DOCTYPE html>
<html>
<head>
    <title>Hello HTML</title>
</head>
<body>
    <h1>Hello HTML!</h1>
    <p>Welcome to your first HTML page.</p>
</body>
</html>
🔤
השלד <head> — מטא-מידע (כותרת, CSS). <body> — התוכן הנראה. תגיות נפוצות: <h1> (כותרת), <p> (פסקה), <button>, <input>, <div> (container).

2. אירועים ומניפולציית DOM

🧭
קונטקסט כדי שדף יהיה אינטראקטיבי צריך להגיב לפעולות המשתמש. הדרך הפשוטה: מאפיין onclick שמפעיל פונקציית JS, שמשנה את ה-DOM.
html · button.html — כפתור שמשנה צבע רקע
<body>
    <h1>Click the Button</h1>
    <button onclick="changeColor()">Click Me</button>

    <script>
        let clicked = false;
        function changeColor() {
            if (!clicked) {
                document.body.style.backgroundColor = "lightblue";
                clicked = true;
            } else {
                document.body.style.backgroundColor = "white";
                clicked = false;
            }
        }
    </script>
</body>
🔤
DOM ו-onclick
  • onclick="changeColor()" — קורא לפונקציה בלחיצה. (ראינו אירועים בהרצאות — זה ה-Listener pattern.)
  • document.body.style.backgroundColor — גישה לאובייקט body ושינוי הסגנון שלו ישירות. זה ה-DOM: ה-JS משנה את העמוד בזמן ריצה.
  • ה-<script> בתוך ה-HTML מריץ JS בדף.

3. JS חיצוני

עדיף להפריד את ה-JS לקובץ נפרד (כמו שהפרדנו CSS) ולקשר אליו עם <script src>:

html · button_file.html
<head>
    <title>External JS</title>
    <script src="button_script.js"></script>
</head>
<body>
    <button onclick="changeColor()">Click Me</button>
</body>
js · button_script.js
let clicked = false;
function changeColor() {
    document.body.style.backgroundColor = clicked ? "white" : "lightblue";
    clicked = !clicked;
}
🎯
הפרדת אחריות HTML למבנה, CSS לעיצוב, JS חיצוני להתנהגות. אותו עיקרון שראינו לאורך כל הקורס — קוד מסודר וניתן לשימוש חוזר.

4. getElementById, innerText ו-value

🧭
קונטקסט כדי לעבוד עם אלמנט ספציפי, נותנים לו id ומאתרים אותו עם document.getElementById. דוגמה: "שעון דפדפן" שקורא מספר מ-input, מאמת אותו, וסופר לאחור על המסך.
html · browser_clock.html
<p id="clock"></p>
<input type="text" id="seconds">
<button onclick="startClock()">Start Clock</button>

<script>
    function startClock() {
        let seconds = document.getElementById("seconds").value;   // קריאת קלט

        if (!isPositiveInteger(seconds)) {                        // ולידציה
            window.alert("Error: Invalid Input!");
            return;
        }
        seconds = parseInt(seconds);

        const oneSecond = 1000;
        for (let i = 1; i < seconds; i++) {
            setTimeout(setClock, oneSecond * i, seconds - i);
        }
        setTimeout(() => {
            document.getElementById("clock").innerText = "Time's Up!";
        }, oneSecond * seconds);
    }

    function setClock(s) {
        document.getElementById("clock").innerText = s;           // שינוי תוכן
    }
</script>
🔤
‎value‎ מול ‎innerText‎
  • element.value — הערך של שדה קלט (<input>). תמיד מחרוזת — לכן parseInt כדי לקבל מספר.
  • element.innerText — הטקסט המוצג בתוך אלמנט (<p>, <div>). שינוי שלו = שינוי מה שרואים.
  • window.alert(...) — חלון הודעה קופץ.
(זה אותו count down מתרגול 4 — אבל הפעם על המסך במקום בקונסול.)

5. ולידציה של קלט

🧭
קונטקסט קלט ממשתמש הוא תמיד מחרוזת ולא תמיד תקין — חייבים לאמת אותו לפני שימוש. פונקציית עזר שבודקת אם מחרוזת היא מספר שלם חיובי:
js · validation.js
function isPositiveInteger(str) {
    if (typeof str !== "string") return false;   // מעבדים רק מחרוזות

    const num = Number(str);                      // המרה למספר
    if (isNaN(num)) return false;                 // ההמרה נכשלה?

    return Number.isInteger(num) && num > 0;       // שלם וחיובי?
}
🔤
כלי הבדיקה
  • typeof x — מחזיר את סוג הטיפוס כמחרוזת ("string", "number"...).
  • Number(str) — ממיר מחרוזת למספר (מחזיר NaN אם נכשל).
  • isNaN(x) — האם הערך הוא "Not a Number".
  • Number.isInteger(x) — האם מספר שלם.

6. טפסים, inputs ו-datalist

🧭
קונטקסט דוגמה מלאה יותר — "רשימת קניות": input עם השלמה אוטומטית (datalist), input מסוג מספר, וכפתור שמוסיף שורה לטבלה דינמית.
html · shopping_list.html (הטופס)
<input id="new-item" list="items">
<datalist id="items">                  <!-- אפשרויות השלמה אוטומטית -->
    <option>T-Shirt</option>
    <option>Shoes</option>
    <option>Pants</option>
</datalist>

<input id="amount" type="number" min="1" value="1">
<button id="add-item-btn" onclick="addItem()">Add Item</button>
<div id="shopping-list-div"><text>No items yet.</text></div>
js · addItem (לוגיקה)
const prices = { 'T-Shirt': 90, 'Shoes': 250, 'Pants': 100 };

function addItem() {
    const item = document.getElementById("new-item").value;
    let amount = document.getElementById("amount").value;

    if (item === "") { alert("Please choose an item."); return; }
    if (!isPositiveInteger(amount)) { alert("Amount must be a positive integer."); return; }

    amount = parseInt(amount);
    const totalPrice = prices[item] * amount;   // חיפוש מחיר במילון
    // ... הוספת שורה לטבלה ב-DOM
}
🔤
אלמנטי טופס
  • <input list="items"> + <datalist id="items"> — שדה עם השלמה אוטומטית מתוך אפשרויות.
  • <input type="number" min="1"> — שדה מספרי עם בורר וערך מינימלי.
  • שמים id על כל אלמנט שצריך לגשת אליו מ-JS.
🎨
בונוס — Bootstrap ו-CSS הדוגמאות (רשימת קניות, מחשבון) משתמשות ב-Bootstrap — ספריית CSS מוכנה שמקשרים אליה ב-<link> (CDN). היא נותנת מחלקות עיצוב מוכנות (card, btn...). אפשר גם CSS משלנו ב-<style> (כמו במחשבון cool_calculator, עם position, border-radius ועוד).

★ סיכום

🎯
מה לקחנו
  • HTML: שלד head/body, תגיות נפוצות, id לזיהוי אלמנטים.
  • אירועים + DOM: onclick מפעיל JS, שמשנה את העמוד דרך document (body.style, getElementById).
  • גישה לאלמנטים: getElementById + .value (קלט, תמיד מחרוזת) / .innerText (טקסט מוצג).
  • ולידציה: typeof, Number, isNaN, Number.isInteger — תמיד לאמת קלט.
  • טפסים: input, datalist, type="number". עיצוב עם Bootstrap או CSS משלנו.
