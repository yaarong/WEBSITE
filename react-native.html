React Native למבחן — אפליקציית מסך יחיד

במבחן, אפליקציית RN של מסך אחד מכסה כמעט הכל: state, fetch מול שרת, טופס הוספה, רשימה, מחיקה ו-props. כאן נבנה בדיוק כזו — רשימת סרטים — ונפרק אותה לשלבים. הקוד נראה זהה ל-React רגיל; רק התגים מתחלפים (<View> במקום <div>, <Text> במקום <p>) והלוגיקה (useState/useEffect/fetch) — זהה לחלוטין.

🧭
המבנה הנכון: App → Home + Card App זעיר — תפקידו רק להחזיר את Home. כל ה"בשר" יושב ב-Home (state + fetch + טופס + רשימה). כל סרט מוצג ע"י MovieCard נפרד. עיקרון מפתח שנראה כאן: ה-state והבקשות (fetch) חיים ב-Home; ה-MovieCard רק מקבל נתונים ופונקציות ב-props.

0️⃣ נקודת הכניסה — index.js + App.js

index.js הוא מה שמריץ את האפליקציה: הוא "רושם" את הקומפוננטה הראשית App כשורש:

javascript · index.js
import { registerRootComponent } from 'expo';
import App from './App';

registerRootComponent(App);

App.js הוא מעטפת של 3 שורות — רק מחזיר את Home:

javascript · App.js
import React from 'react';
import Home from './Home';

export default function App() {
  return <Home />;
}
🔤
למה App כל כך קטן? זהו ה"שער". ב-web זה היה App עם ה-Router שמחליט איזה מסך להציג. כאן — מסך יחיד, אז אין Router; פשוט מחזירים את <Home />. שים לב ל-export default (כך ש-App.js ו-index.js יכולים לייבא אותו).

1️⃣ Home.js — הקומפוננטה הגדולה

זה הלב. נפרק אותו לחלקים, ואז נראה אותו במלואו (כולל המחיקה).

1.1 ייבוא — הרכיבים של RN

javascript · imports
import React, { useState, useEffect } from 'react';
import { View, TextInput, Button, FlatList, StyleSheet } from 'react-native';
import MovieCard from './MovieCard';
📦
חובה לייבא כל רכיב RN ב-RN אין תגי HTML מובנים — כל רכיב (View, Text, TextInput, Button, FlatList, StyleSheet) מיובא במפורש מ-'react-native'. שכחת לייבא → שגיאה.

1.2 כתובת השרת — BASE_URL

javascript
const BASE_URL = 'http://10.0.2.2:3000';
🌐
10.0.2.2 — לא localhost! RN רץ באמולטור/מכשיר נפרד מהשרת, ולכן מתחבר ב-כתובת מלאה (לא נתיב יחסי כמו ב-web). 10.0.2.2 = מכונת ה-host כפי שנראית מאמולטור אנדרואיד (ב-iOS simulator זה localhost; במכשיר אמיתי — ה-IP ברשת).

1.3 ה-state — רשימה + שדות הטופס

javascript
const [movies, setMovies] = useState([]);   // הרשימה מהשרת
const [title, setTitle]   = useState('');   // שדות הטופס
const [genre, setGenre]   = useState('');
const [year,  setYear]    = useState('');
🧠
4 חתיכות state אחת לרשימה (movies, מתחיל ריק []), ואחת לכל שדה בטופס. כל הקלדה תעדכן את ה-state המתאים, וכל setState גורם ל-React לצייר מחדש.

1.4 טעינת הרשימה — loadMovies (GET)

javascript
async function loadMovies() {
  const res  = await fetch(`${BASE_URL}/api/movies`);
  const data = await res.json();
  setMovies(data);   // שומר את התוצאה ב-state → ציור מחדש
}

1.5 טעינה פעם אחת — useEffect

javascript
useEffect(() => { loadMovies(); }, []);   // [] → רץ פעם אחת, בטעינת המסך
⚠️
למה useEffect ולא לקרוא ל-fetch ישר? אם נקרא ל-loadMovies() בגוף הקומפוננטה — הוא ירוץ בכל ציור מחדש (אינסוף). ה-[] בסוף אומר "רוץ פעם אחת בלבד, כשהמסך נטען".

1.6 הוספת סרט — handleAdd (POST)

javascript
async function handleAdd() {
  await fetch(`${BASE_URL}/api/movies`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ title, genre, year: Number(year) })
  });
  setTitle(''); setGenre(''); setYear('');   // מרוקן את הטופס
  loadMovies();                               // רענון הרשימה
}
🔢
Number(year) ערך מ-TextInput תמיד מחרוזת. אם השרת מצפה ל-מספר (year), ממירים עם Number(...) לפני השליחה. אחרי הוספה: מרוקנים את הטופס וטוענים מחדש כדי שהרשימה תתעדכן.

1.7 מחיקת סרט — handleDelete (DELETE)

javascript
async function handleDelete(id) {
  await fetch(`${BASE_URL}/api/movies/${id}`, { method: 'DELETE' });
  loadMovies();   // רענון הרשימה אחרי מחיקה
}
🎯
המחיקה חיה ב-Home — לא ב-Card ה-fetch של המחיקה נשאר ב-Home כי שם ה-state והרשימה (רק Home יכול לקרוא ל-loadMovies). מקבל את ה-id שצריך למחוק, פונה לשרת, ומרענן. את ה-handleDelete נעביר ל-Card כ-prop (נראה בשלב 1.8 ובמחלקה 3).

1.8 ה-JSX — טופס + רשימה

jsx · return
return (
  <View style={styles.container}>
    <TextInput style={styles.input} placeholder="Title" value={title} onChangeText={setTitle} />
    <TextInput style={styles.input} placeholder="Genre" value={genre} onChangeText={setGenre} />
    <TextInput style={styles.input} placeholder="Year"  value={year}  onChangeText={setYear} keyboardType="numeric" />
    <Button title="Add Movie" onPress={handleAdd} />

    <FlatList
      data={movies}
      keyExtractor={(item) => item._id}
      renderItem={({ item }) => <MovieCard movie={item} onDelete={handleDelete} />}
    />
  </View>
);
🔤
FlatList — הרשימה של RN
  • data={movies} — מערך הנתונים.
  • keyExtractor={(item) => item._id} — מזהה ייחודי לכל שורה (כמו key ב-web; כאן ה-_id ממונגו).
  • renderItem={({ item }) => ...} — איך לצייר כל פריט. שים לב לפירוק { item }. כאן מעבירים ל-MovieCard גם את movie וגם את onDelete.

1.9 העיצוב — StyleSheet

javascript
const styles = StyleSheet.create({
  container: { flex: 1, padding: 20 },
  input: { borderWidth: 1, padding: 8, marginBottom: 8 }
});
🎨
עיצוב ב-RN — style, לא className אין main.css ואין className. מגדירים אובייקט עם StyleSheet.create ומחילים עם style={styles.x}. השמות ב-camelCase (marginBottom, לא margin-bottom) והערכים מספרים בלי px.

1.10 Home.js המלא (כולל מחיקה)

javascript · Home.js — מלא
import React, { useState, useEffect } from 'react';
import { View, TextInput, Button, FlatList, StyleSheet } from 'react-native';
import MovieCard from './MovieCard';

const BASE_URL = 'http://10.0.2.2:3000';

export default function Home() {
  const [movies, setMovies] = useState([]);
  const [title, setTitle] = useState('');
  const [genre, setGenre] = useState('');
  const [year, setYear]   = useState('');

  async function loadMovies() {
    const res = await fetch(`${BASE_URL}/api/movies`);
    const data = await res.json();
    setMovies(data);
  }

  useEffect(() => { loadMovies(); }, []);

  async function handleAdd() {
    await fetch(`${BASE_URL}/api/movies`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ title, genre, year: Number(year) })
    });
    setTitle(''); setGenre(''); setYear('');
    loadMovies();
  }

  async function handleDelete(id) {
    await fetch(`${BASE_URL}/api/movies/${id}`, { method: 'DELETE' });
    loadMovies();
  }

  return (
    <View style={styles.container}>
      <TextInput style={styles.input} placeholder="Title" value={title} onChangeText={setTitle} />
      <TextInput style={styles.input} placeholder="Genre" value={genre} onChangeText={setGenre} />
      <TextInput style={styles.input} placeholder="Year"  value={year}  onChangeText={setYear} keyboardType="numeric" />
      <Button title="Add Movie" onPress={handleAdd} />

      <FlatList
        data={movies}
        keyExtractor={(item) => item._id}
        renderItem={({ item }) => <MovieCard movie={item} onDelete={handleDelete} />}
      />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20 },
  input: { borderWidth: 1, padding: 8, marginBottom: 8 }
});

2️⃣ MovieCard.js — סרט בודד + מחיקה

קומפוננטה קטנה שמקבלת ב-props את movie (מה להציג) ואת onDelete (הפונקציה ללחיצה). כל כרטיס מוחק את עצמו — אבל רק קורא לפונקציה שקיבל מ-Home:

javascript · MovieCard.js — מלא
import React from 'react';
import { View, Text, Button, StyleSheet } from 'react-native';   // + Button

export default function MovieCard({ movie, onDelete }) {          // + onDelete
  return (
    <View style={styles.card}>
      <Text style={styles.title}>{movie.title}</Text>
      <Text>{movie.genre} · {movie.year}</Text>
      <Button title="Delete" onPress={() => onDelete(movie._id)} />
    </View>
  );
}

const styles = StyleSheet.create({
  card: { padding: 12, borderWidth: 1, marginBottom: 8 },
  title: { fontWeight: 'bold', fontSize: 18 }
});
⭐
שתי נקודות קריטיות בכרטיס
  • כל טקסט חייב להיות בתוך <Text> — אין "טקסט עירום" ב-RN (מפיל הרבה).
  • onPress={() => onDelete(movie._id)} — עוטפים בחץ כדי להעביר את ה-id. אם נכתוב onPress={onDelete(movie._id)} (בלי החץ) — זה יקרא לפונקציה מיד בציור, לא בלחיצה!
🔤
{ movie, onDelete } — destructuring של props הקומפוננטה מקבלת אובייקט props, ואנחנו שולפים ממנו ישר את השדות. movie = הנתונים להצגה; onDelete = פונקציה שהאב (Home) נתן. הכרטיס לא יודע (ולא צריך לדעת) איך המחיקה עובדת — הוא רק לוחץ.

3️⃣ העיקרון — מי מחזיק state, מי מבצע, מי לוחץ

זה התבנית החשובה ביותר ב-React/RN, ומה שחמי בודק: ה-state חי במקום אחד (האב), והבן רק מקבל נתונים ופונקציות ב-props.

Home (האב)MovieCard (הבן)
מחזיק את ה-state (movies)✅ כן❌ לא
מבצע את ה-fetch✅ handleDelete, handleAdd❌ לא
מרענן את הרשימה (loadMovies)✅ כן❌ לא
מקבל נתונים/פונקציה ב-props—✅ movie + onDelete
לוחץ (מפעיל את הפונקציה)—✅ onPress={() => onDelete(movie._id)}
🔁
למה לא לעשות את ה-fetch בתוך הכרטיס? כי אחרי המחיקה צריך לרענן את הרשימה — וה-state של הרשימה (setMovies) חי ב-Home, לא בכרטיס. אם הכרטיס היה מוחק לבד, הוא לא היה יכול לעדכן את הרשימה של האב. לכן: הבן מודיע לאב ("מחק את ה-id הזה"), והאב מבצע ומרענן. בדיוק אותו עיקרון כמו הוספה.

4️⃣ הזרימה המלאה של האפליקציה

  1. האפליקציה עולה: index.js → App → Home.
  2. Home מצייר את הטופס והרשימה (ריקה), ואז useEffect מריץ loadMovies פעם אחת.
  3. loadMovies עושה GET /api/movies → setMovies(data) → הרשימה מצוירת מחדש, כל סרט כ-MovieCard.
  4. המשתמש ממלא טופס ולוחץ Add Movie → handleAdd עושה POST → מרוקן טופס → loadMovies מרענן.
  5. המשתמש לוחץ Delete בכרטיס → הכרטיס קורא ל-onDelete(id) → Home עושה DELETE → loadMovies מרענן.
🔗
צד השרת (Backend) זה אותו שרת Express + MongoDB מהתרגילים: GET /api/movies (כל הסרטים), POST /api/movies (יצירה), DELETE /api/movies/:id (מחיקה). ה-RN הוא רק ה-Frontend שמדבר איתו ב-fetch.
