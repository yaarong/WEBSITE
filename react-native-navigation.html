React Native עם כמה מסכים — Stack Navigator

זה מקביל כמעט אחד-לאחד ל-React Router שכבר ראינו — רק עם השמות של React Navigation. אותו רעיון בדיוק: ה-App הופך ממסך-יחיד למפת ניתוב, וכל מסך הוא קומפוננטה רגילה. אם הבנת את ה-Router של ה-web — אתה כבר יודע את זה; צריך רק להחליף שמות.

🧭
קונטקסט — אם אתה מכיר React Router, אתה כבר כאן כל מה שלמדנו ב-React (web) — Router תקף: App = מפת ניתוב, כל מסך עצמאי, מ-2 מסכים ל-N זה אותו דפוס. ההבדל היחיד הוא שמות הרכיבים (NavigationContainer במקום BrowserRouter וכו') ושהמפתח הוא שם ("AddPlant") במקום URL (/add).

1️⃣ מה משתנה ב-App — נכנס Stack Navigator

בדיוק כמו ב-web, ה-App הופך ממסך-יחיד למפת ניתוב:

javascript · App.js
import React from 'react';
import { NavigationContainer } from '@react-navigation/native';
import { createStackNavigator } from '@react-navigation/stack';
import Home from './Home';
import AddPlant from './AddPlant';

const Stack = createStackNavigator();

export default function App() {
  return (
    <NavigationContainer>
      <Stack.Navigator>
        <Stack.Screen name="Home"     component={Home} />
        <Stack.Screen name="AddPlant" component={AddPlant} />
      </Stack.Navigator>
    </NavigationContainer>
  );
}

ההקבלה ל-web — זה כל מה שצריך לזכור:

RN (React Navigation)web (React Router)תפקיד
NavigationContainerBrowserRouterעוטף הכל
Stack.NavigatorSwitchמנהל את ערימת המסכים
<Stack.Screen name="X" component={Y}/><Route path="/x" component={Y}/>ממפה שם→מסך
🔤
ההבדל היחיד: שם במקום URL ב-web המפתח של כל מסך הוא URL (/add); ב-RN זה שם ("AddPlant") — כי לאפליקציית מובייל אין שורת כתובת. זהו. כל השאר זהה במהות.
📚
למה "Stack"? createStackNavigator בונה ניווט מסוג ערימה (stack): כל מסך חדש "נדחף" מעל הקודם (עם אנימציית מעבר וכפתור "חזור" אוטומטי בראש). זה הדפוס הנפוץ ביותר במובייל — בדיוק כמו שפתיחת מסך חדש בטלפון "דוחפת" אותו מעל הנוכחי, ולחיצה על "אחורה" מסירה אותו.

2️⃣ איך מנווטים — ה-prop navigation

כאן יש הקלה לעומת ה-web: ב-RN, כל קומפוננטת מסך מקבלת אוטומטית prop בשם navigation — לא צריך hook כמו useHistory. פשוט שולפים אותו וקוראים לו:

javascript · ניווט מתוך מסך
function Home({ navigation }) {     // ← navigation מגיע אוטומטית כ-prop
  // ...
  <Button title="Add Plant" onPress={() => navigation.navigate('AddPlant')} />
}
🔤
navigation.navigate('AddPlant') "עבור למסך הזה". זה השקול ל-history.push('/add') ול-<Link> ביחד — גם ניווט מהקוד וגם הדרך הסטנדרטית לעבור מסך. שים לב: מעבירים את שם המסך (כפי שהוגדר ב-<Stack.Screen name="...">), לא URL.
🎁
למה לא צריך hook? ה-Stack.Navigator "מזריק" את ה-prop navigation לכל קומפוננטה שרשומה כ-<Stack.Screen> — אוטומטית. לכן Home ו-AddPlant מקבלים אותו "חינם". (ב-web היינו צריכים useHistory() כי אין מנגנון כזה.)

2.1 העברת נתון בין מסכים (להכללה)

אם רוצים להעביר מידע למסך הבא — שמים אותו כאובייקט שני ב-navigate, ובמסך היעד שולפים מה-prop route:

javascript · העברת פרמטר
// מסך שולח:
navigation.navigate('Details', { id: plant._id });

// מסך מקבל:
function Details({ route }) {        // ← route מגיע אוטומטית כ-prop
  const { id } = route.params;       // שולף את מה שנשלח
  // ... fetch לפי id ...
}
🔗
ההקבלה — route.params מול useParams navigation.navigate('Details', { id }) + route.params.id ב-RN = <Link to={`/plants/${id}`}> + useParams() ב-web. (זה ה-putExtra/params בניב RN.) למבחן ברמה הזו — בד"כ לא צריך, אבל טוב לדעת.

3.3 Home.js — רשימה + ניווט למסך ההוספה

javascript · Home.js
import React, { useState, useEffect } from 'react';
import { View, Button, FlatList, StyleSheet } from 'react-native';
import PlantCard from './PlantCard';

const BASE_URL = 'http://10.0.2.2:3000';

export default function Home({ navigation }) {
  const [plants, setPlants] = useState([]);

  async function loadPlants() {
    const res = await fetch(`${BASE_URL}/api/plants`);
    const data = await res.json();
    setPlants(data);
  }

  useEffect(() => { loadPlants(); }, []);

  async function handleDelete(id) {
    await fetch(`${BASE_URL}/api/plants/${id}`, { method: 'DELETE' });
    loadPlants();
  }

  return (
    <View style={styles.container}>
      <Button title="+ Add Plant" onPress={() => navigation.navigate('AddPlant')} />
      <FlatList
        data={plants}
        keyExtractor={(item) => item._id}
        renderItem={({ item }) => <PlantCard plant={item} onDelete={handleDelete} />}
      />
    </View>
  );
}

const styles = StyleSheet.create({ container: { flex: 1, padding: 20 } });
🔁
המסך הזה כבר מוכר לך זה בדיוק ה-Home של RN ממסך יחיד (state + useEffect + fetch + FlatList + מחיקה דרך prop) — התוספת היחידה: { navigation } ב-props, וכפתור שקורא ל-navigation.navigate('AddPlant'). הטופס עבר למסך נפרד.

3.4 AddPlant.js — טופס, ואחרי POST → חזרה ל-Home

javascript · AddPlant.js
import React, { useState } from 'react';
import { View, TextInput, Button, StyleSheet } from 'react-native';

const BASE_URL = 'http://10.0.2.2:3000';

export default function AddPlant({ navigation }) {
  const [name, setName]       = useState('');
  const [species, setSpecies] = useState('');
  const [price, setPrice]     = useState('');

  async function handleAdd() {
    await fetch(`${BASE_URL}/api/plants`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ name, species, price: Number(price) })
    });
    navigation.navigate('Home');     // ← חזרה ל-Home אחרי ההוספה
  }

  return (
    <View style={styles.container}>
      <TextInput style={styles.input} placeholder="Name"    value={name}    onChangeText={setName} />
      <TextInput style={styles.input} placeholder="Species" value={species} onChangeText={setSpecies} />
      <TextInput style={styles.input} placeholder="Price"   value={price}   onChangeText={setPrice} keyboardType="numeric" />
      <Button title="Save" onPress={handleAdd} />
    </View>
  );
}

const styles = StyleSheet.create({
  container: { flex: 1, padding: 20 },
  input: { borderWidth: 1, padding: 8, marginBottom: 8 }
});
🎯
אותה לוגיקה כמו ב-web אחרי ה-POST פשוט navigation.navigate('Home') — בדיוק כמו history.push('/') ב-web. כשחוזרים ל-Home, ה-useEffect שלו ירוץ ויטען את הרשימה מחדש, כך שהצמח החדש יופיע. שים לב לאירועי RN: onChangeText (לא onChange) ו-onPress (לא onClick).

3.5 PlantCard.js — ללא שינוי

javascript · PlantCard.js
import React from 'react';
import { View, Text, Button, StyleSheet } from 'react-native';

export default function PlantCard({ plant, onDelete }) {
  return (
    <View style={styles.card}>
      <Text style={styles.title}>{plant.name}</Text>
      <Text>{plant.species} · ${plant.price}</Text>
      <Button title="Delete" onPress={() => onDelete(plant._id)} />

  );
}

const styles = StyleSheet.create({
  card: { padding: 12, borderWidth: 1, marginBottom: 8 },
  title: { fontWeight: 'bold', fontSize: 18 }

🧩
הכרטיס לא יודע דבר על ניווט בדיוק כמו ב-web — הוא מקבל plant ו-onDelete ב-props ולוחץ. אין בו navigation כי הוא לא מסך (הוא לא רשום כ-<Stack.Screen>). הניווט הוא עניין של המסכים בלבד.

🎯
המשפט אחד לזכור אותו רעיון בדיוק, שמות אחרים. App = מפת ניתוב (NavigationContainer + Stack.Navigator + Stack.Screen-ים). מנווטים עם navigation.navigate('Name') (מגיע אוטומטית כ-prop). מסך נוסף = עוד <Stack.Screen> + עוד קובץ.
🪤
מלכודות RN שעדיין תקפות גם כאן: כל טקסט בתוך <Text>, אירועים onPress/onChangeText (לא onClick/onChange), עיצוב עם style={styles.x} + StyleSheet.create, וכתובת שרת מלאה (10.0.2.2). הניווט לא משנה את הכללים האלה.
