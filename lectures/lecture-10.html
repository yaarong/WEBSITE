הרצאה 10

React Native — בונים אפליקציית מובייל עם React (ו-Expo)

בהרצאה 8 ראינו את React לבניית ממשקי web, ודיברנו על הדילמה Native מול Cross-Platform. עכשיו נשלב את השניים: React Native מאפשר לכתוב קוד React — אותם רעיונות בדיוק (components, props, state, JSX) — ולקבל אפליקציית מובייל אמיתית ל-iOS ול-Android. נשתמש ב-Expo (סביבת עבודה שמקלה על הכל), ונבנה מחדש את אתר ה-Foo News — הפעם כאפליקציית מובייל.

🗺️
קונטקסט כללי — למה זה מרגש זוכרים את הטבלה מהרצאה 8 (Web / Mobile / Desktop), ואת ה-cross-platform ("write once, run anywhere")? React Native הוא בדיוק זה: אתם כבר יודעים React מהרצאה 8 — וכמעט בלי ללמוד שפה חדשה, אתם בונים אפליקציית מובייל. ההבדל המרכזי: במקום תגיות HTML (div, span) משתמשים ברכיבי native (View, Text), ובמקום CSS משתמשים ב-StyleSheet ו-Flexbox. כל השאר — מוכר.

חלק א' · מה זה React Native

1. React Native ו-Expo — הרעיון

🧭
קונטקסט React Native היא מסגרת (framework) של Meta שמאפשרת לכתוב אפליקציות מובייל native בעזרת React. כותבים JavaScript + JSX, ו-React Native מתרגם את הרכיבים לרכיבי UI אמיתיים של מערכת ההפעלה (לא דף web בתוך אפליקציה!). קוד אחד → אפליקציה ל-Android, ל-iOS, ואפילו ל-web. זו גישת ה-cross-platform מהרצאה 8.
📦
מה זה Expo Expo היא ערכת כלים (toolchain) שעוטפת את React Native והופכת את הפיתוח לקל: הקמת פרויקט בפקודה אחת, הרצה על אמולטור/מכשיר אמיתי בלי הגדרות מסובכות, ספריות מובנות (מצלמה, מיקום, התראות), וכלי ניווט. זה ה-"create-react-app" של עולם המובייל.
🔤
React מול React Native — מה זהה ומה שונה
  • זהה: components (פונקציות שמחזירות JSX), props, state (useState, useEffect), הרעיון "UI הוא פונקציה של ה-state".
  • שונה: אין div/span/button — יש View/Text/Button. אין CSS — יש StyleSheet ו-Flexbox. אין דפדפן — יש מכשיר.

2. הקמת פרויקט — create-expo-app

פותחים פרויקט בפקודה אחת (בדיוק כמו create-react-app מהרצאה 8):

bash · יצירה והרצה
npx create-expo-app MyTodoApp --template blank

cd MyTodoApp
npm run android   # הרצה על אמולטור/מכשיר אנדרואיד
npm run ios       # הרצה על סימולטור iOS
npm run web       # הרצה בדפדפן

מבנה הפרויקט הבסיסי:

text · מבנה תיקיות
MyTodoApp/
├── assets/            (תמונות, אייקונים)
├── node_modules/
├── App.js             ← הרכיב הראשי
├── app.json           (קונפיגורציה של Expo)
├── index.js           (נקודת הכניסה)
└── package.json

index.js — נקודת הכניסה. רושמת את App כרכיב השורש:

js · index.js
import { registerRootComponent } from 'expo';
import App from './App';

registerRootComponent(App);
🔤
המקבילה ל-web registerRootComponent(App) הוא המקביל של ReactDOM.createRoot(...).render(<App />) מהרצאה 8 — רק שכאן אין #root ב-HTML; ה"שורש" הוא המסך של המכשיר.

חלק ב' · רכיבים ועיצוב

3. Core Components — View, Text, Image, Button, TextInput

🧭
קונטקסט ב-React Native אין תגיות HTML. במקום זה יש רכיבי ליבה שמתורגמים לרכיבי native. הם מקבילים ישירות למה שכבר ראינו ב-web ובאנדרואיד (הרצאה 8):
React NativeWeb (HTML)Androidמה זה
<View><div>LinearLayoutcontainer לסידור רכיבים
<Text><span>/<p>TextViewהצגת טקסט
<Image><img>ImageViewתמונה
<Button><button>Buttonכפתור
<TextInput><input>EditTextשדה קלט

ה-App.js הבסיסי שמגיע מ-Expo:

jsx · App.js
import { StatusBar } from 'expo-status-bar';
import { StyleSheet, Text, View } from 'react-native';

export default function App() {
    return (
        <View style={styles.container}>
            <Text>Open up App.js to start working on your app!</Text>
            <StatusBar style="auto" />
        </View>
    );
}

const styles = StyleSheet.create({
    container: {
        flex: 1,
        backgroundColor: '#fff',
        alignItems: 'center',
        justifyContent: 'center',
    },
});
🪤
הכלל הכי חשוב — כל טקסט חייב להיות בתוך <Text> ב-web אפשר לכתוב טקסט ישירות בתוך <div>. ב-React Native — אסור. כל מחרוזת טקסט חייבת להיות עטופה ב-<Text>, אחרת תקבלו שגיאה. <View> הוא רק container — הוא לא יודע להציג טקסט בעצמו.

דוגמה עשירה יותר — View עם Text, Button, Image ו-TextInput:

jsx · App.js — כמה רכיבים
import { StyleSheet, Text, View, Image, Button, TextInput } from 'react-native';
import Cake from './assets/foo.jpg';

export default function App() {
    return (
        <View>
            <Text>Hello world</Text>
            <Button title="Click me!" />
            <View>
                <Image source={Cake} style={{ width: 200, height: 200 }} />
                <TextInput defaultValue="Text me!" />
            </View>
        </View>
    );
}
🔤
נקודות תחביר
  • <Button title="..." /> — הטקסט של הכפתור עובר ב-prop title, לא בתוך התגית.
  • <Image source={Cake} /> — מקור התמונה הוא import של הקובץ, לא נתיב מחרוזת.
  • style={{ width: 200, height: 200 }} — סגנון inline הוא אובייקט JS (שימו לב לסוגריים הכפולים: חיצוני ל-JSX, פנימי לאובייקט).

4. StyleSheet ו-Flexbox

🧭
קונטקסט אין קובצי CSS. במקום זאת, מגדירים סגנונות כאובייקטי JavaScript עם StyleSheet.create. השמות דומים ל-CSS אבל ב-camelCase (backgroundColor במקום background-color), והפריסה מבוססת כולה על Flexbox.
jsx · App.js עם StyleSheet
export default function App() {
    return (
        <View style={styles.container}>
            <Text>Hello world</Text>
            <View style={{ backgroundColor: '#1f1c1c', padding: 10 }}>
                <Image source={Cake} style={{ width: 200, height: 200 }} />
            </View>
            <TextInput defaultValue="Text me!" style={styles.input} />
        </View>
    );
}

const styles = StyleSheet.create({
    container: {
        padding: 20,
        flex: 1,
        justifyContent: 'center',
        alignItems: 'center',
    },
    input: {
        height: 40,
        borderColor: 'gray',
        borderWidth: 1,
        marginTop: 10,
        paddingHorizontal: 10,
    },
});
🔤
StyleSheet ו-Flexbox
  • StyleSheet.create({...}) — יוצר אובייקט סגנונות. ניגשים אליו עם styles.container ומעבירים ל-style={...}.
  • flex: 1 — "תפוס את כל המקום הפנוי". זהו עקרון ה-Flexbox המרכזי.
  • justifyContent (יישור על הציר הראשי) ו-alignItems (על הציר המשני) — בדיוק כמו Flexbox ב-CSS, רק שכאן זו ברירת המחדל של כל View (ואין צורך ב-display: flex).
  • שמות ב-camelCase: backgroundColor, borderWidth, paddingHorizontal. ערכים מספריים הם בפיקסלים (בלי "px").
🔬
מתחת למכסה כשמריצים על web, אפשר לפתוח את ה-DevTools ולראות שה-<View> הפך ל-<div class="css-view-...">. ב-mobile, לעומת זאת, אותו <View> הופך לרכיב native אמיתי של מערכת ההפעלה. אותו קוד, פלט שונה לכל פלטפורמה — זה כל הקסם של cross-platform.

חלק ג' · ניווט (Expo Router)

6. ניווט מבוסס-קבצים — Stack, Tabs, useRouter

🧭
קונטקסט אפליקציה היא יותר ממסך אחד — צריך ניווט בין מסכים. Expo Router משתמש בניווט מבוסס-קבצים: כל קובץ בתיקיית app/ הופך אוטומטית למסך, לפי שם הקובץ. (רעיון מוכר? כן — בדיוק כמו ש-routes ב-Express מיפו URL לפעולה, רק שכאן ה-URL נגזר ממבנה התיקיות.)

מתקינים את הספרייה ומגדירים את נקודת הכניסה ב-package.json:

bash · התקנה
npx expo install expo-router react-native-safe-area-context \
    react-native-screens expo-linking expo-constants expo-status-bar

# ב-package.json:  "main": "expo-router/entry"

Stack — ניווט מסך-על-מסך (כמו ערימת ה-Activities שנראה בהרצאה 11). הקובץ app/_layout.js מגדיר את מבנה הניווט:

jsx · app/_layout.js
import { Stack } from 'expo-router';

export default function Layout() {
    return <Stack />;
}

שני מסכים — app/index.js (מסך הבית) ו-app/foo.js — וניווט ביניהם עם useRouter:

jsx · app/index.js
import { View, Text, Button } from 'react-native';
import { useRouter } from 'expo-router';

export default function Home() {
    const router = useRouter();
    return (
        <View>
            <Text>Home Screen</Text>
            <Button title="Go to foo" onPress={() => router.push('/foo')} />
        </View>
    );
}
jsx · app/foo.js
import { View, Text, Button } from 'react-native';
import { useRouter } from 'expo-router';

export default function Foo() {
    const router = useRouter();
    return (
        <View>
            <Text>Foo Screen</Text>
            <Button title="Go back" onPress={() => router.back()} />
        </View>
    );
}
🔤
פעולות ניווט
  • useRouter() — Hook שמחזיר את אובייקט ה-router.
  • router.push('/foo') — דוחף מסך חדש לערימה (אפשר לחזור אחורה).
  • router.back() — חוזר למסך הקודם.
  • router.replace('/(tabs)') — מחליף את המסך הנוכחי (בלי אפשרות לחזור — שימושי אחרי login).
  • onPress — ב-React Native אין onClick אלא onPress (כי זה מגע, לא קליק). שאר העיקרון זהה — מעבירים פונקציה, לא קוראים לה.
📑
Tabs ו-SafeAreaView מלבד Stack יש גם Tabs (סרגל לשוניות תחתון), וקבוצות נתיבים בסוגריים כמו (tabs)/ שלא מופיעות ב-URL. SafeAreaView מבטיח שהתוכן לא יוסתר ע"י ה-notch/שורת הסטטוס. נראה את כולם בפעולה כשנבנה את Foo News.

8. רכיב לשימוש חוזר — AppButton

🧭
קונטקסט הכפתור המובנה <Button> מוגבל בעיצוב. לכן בונים כפתור משלנו עם TouchableOpacity (View שמגיב למגע ועושה אפקט שקיפות בלחיצה) — ומקבלים title ו-onPress כ-props (בדיוק כמו props בהרצאה 8).
jsx · components/MainButton.js
import { Text, TouchableOpacity } from 'react-native';
import { styles } from '../styles/MainButton.styles';

export default function AppButton({ title, onPress }) {
    return (
        <TouchableOpacity style={styles.button} onPress={onPress}>
            <Text style={styles.buttonText}>{title}</Text>
        </TouchableOpacity>
    );
}
js · styles/MainButton.styles.js
import { StyleSheet } from 'react-native';
import { COLORS, FONTS, SPACING } from './Theme';

export const styles = StyleSheet.create({
    button: {
        height: FONTS.inputHeight,
        backgroundColor: '#1C1C1E',
        borderRadius: 12,
        justifyContent: 'center',
        alignItems: 'center',
        marginTop: SPACING.md,
    },
    buttonText: {
        color: COLORS.background,
        fontSize: FONTS.standard,
        fontWeight: FONTS.semiBold,
    },
});

9. מסכים — Layout ראשי, login, ו-Tabs

app/_layout.js — ה-Stack הראשי. מסך login ואז קבוצת ה-tabs, שניהם בלי header:

jsx · app/_layout.js
import { Stack } from 'expo-router';

export default function RootLayout() {
    return (
        <Stack>
            <Stack.Screen name="login"  options={{ headerShown: false }} />
            <Stack.Screen name="(tabs)" options={{ headerShown: false }} />
        </Stack>
    );
}

app/login.js — מסך התחברות. אחרי "Let's go!" עוברים ל-tabs עם replace (כדי שלא יחזרו ל-login ב-back):

jsx · app/login.js
import { View, Text, TextInput } from 'react-native';
import { useRouter } from 'expo-router';
import { styles } from '../styles/login.styles';
import AppButton from '../components/MainButton';

export default function Login() {
    const router = useRouter();
    return (
        <View style={styles.container}>
            <Text style={styles.logo}>Foo News</Text>
            <Text style={styles.subtitle}>Stay update to date with the latest foo</Text>

            <TextInput placeholder="Email address" style={styles.input} />
            <TextInput placeholder="Password" style={styles.input} secureTextEntry />

            <AppButton title="Let's go!" onPress={() => router.replace('/(tabs)')} />
        </View>
    );
}
🔤
נקודות
  • placeholder — טקסט רמז ב-input (כמו ב-HTML). secureTextEntry — מסתיר את הסיסמה.
  • <AppButton title="..." onPress={...} /> — משתמשים ברכיב שלנו עם props. הרכיב לשימוש חוזר משתלם מיד.

app/(tabs)/_layout.js — סרגל הלשוניות התחתון, עם אייקון לכל לשונית:

jsx · app/(tabs)/_layout.js
import { Tabs } from 'expo-router';
import { View, Text, Image } from 'react-native';
import { SafeAreaView } from 'react-native-safe-area-context';
import { styles } from '../../styles/tabs_layout.styles';

export default function TabsLayout() {
    return (
        <View style={styles.wrapper}>
            <SafeAreaView style={styles.headerContainer} edges={['top']}>
                <View style={styles.headerContent}>
                    <Text style={styles.logoText}>Foo News</Text>
                </View>
            </SafeAreaView>

            <Tabs screenOptions={{ headerShown: false }}>
                <Tabs.Screen name="index"  options={{ tabBarLabel: 'News feed' }} />
                <Tabs.Screen name="create" options={{ tabBarLabel: 'Create article' }} />
            </Tabs>
        </View>
    );
}

10. News Feed — FlatList ו-ArticleCard

🧭
קונטקסט — FlatList בהרצאה 8 רינדרנו רשימה עם articles.map(...). ב-mobile עדיף FlatList: רכיב שמרנדר רשימות גדולות ביעילות — הוא מצייר רק את הפריטים הנראים על המסך ("virtualization"). זה ההבדל מ-map: ביצועים על רשימות ארוכות (בדיוק כמו RecyclerView באנדרואיד, שנראה בהרצאה 11).

import { useState } from 'react';
import { FlatList, View } from 'react-native';
import ArticleCard from '../../components/Card.js';
import { articlesList } from '../../data/articles.js';
import { styles } from '../../styles/index.styles';

export default function NewsFeed() {

    return (
        <View style={styles.container}>
            <FlatList
                data={articles}
                renderItem={({ item }) => <ArticleCard {...item} />}
                keyExtractor={item => item.id}
                contentContainerStyle={{ padding: 5 }}
            />
        </View>
    );
}
🔤
props של FlatList
  • data — מערך הנתונים.
  • renderItem={({ item }) => ...} — פונקציה שמקבלת כל פריט ומחזירה רכיב. שימו לב ל-{...item} — spread של props, בדיוק כמו בהרצאה 8.
  • keyExtractor — מחזיר מזהה ייחודי לכל פריט (React צריך key לרשימות, כדי לעקוב אחרי שינויים ביעילות).

components/Card.js — רכיב הכרטיס. לחיצה מנווטת לכתבה עם router.push:

jsx · components/Card.js
import { View, Text, Image, TouchableOpacity } from 'react-native';

11. מסך create ומסך הכתבה — נתיב דינמי [id]

🧭
קונטקסט — נתיב דינמי לכתבה צריך מסך פר-id. ב-Expo Router, קובץ בשם [id].js הוא נתיב דינמי — ה-[id] תופס כל ערך מה-URL (בדיוק כמו /:id ב-Express מהרצאה 7!). קוראים אותו עם useLocalSearchParams.

app/(tabs)/create.js — יצירת כתבה:

jsx · app/(tabs)/create.js
import { View, TextInput } from 'react-native';
import { useRouter } from 'expo-router';
import AppButton from '../../components/MainButton';
import { styles } from '../../styles/create.styles';

export default function Create() {
    const router = useRouter();
    return (
        <View style={styles.container}>
            <TextInput placeholder="Title"   style={styles.input} />
            <TextInput placeholder="Content" style={[styles.input, { height: 100 }]} multiline />
            <AppButton title="Create" onPress={() => router.replace('/')} />
        </View>
    );
}
🔤
סגנונות מרובים ו-multiline style={[styles.input, { height: 100 }]} — אפשר להעביר מערך של סגנונות, והם מתמזגים (האחרון גובר). multiline — מאפשר כמה שורות ב-TextInput.

app/article/[id].js — מסך הכתבה הבודדת:

jsx · app/article/[id].js
import { View, Text, ScrollView, Image } from 'react-native';
import { useLocalSearchParams } from 'expo-router';
import { styles } from '../../styles/[id].styles';
import { articlesList } from '../../data/articles';

export default function ArticleInsideTabs() {
    const { id } = useLocalSearchParams();
    const article = articlesList.filter(one => one.id == id)[0];

    return (
        <ScrollView style={styles.container}>
            <Image source={article.image} style={styles.image} />
            <View style={styles.content}>
                <Text style={styles.title}>{article.title}</Text>
                <Text style={styles.subtitle}>{article.author}</Text>
                <Text style={styles.body}>{article.content}</Text>
            </View>
        </ScrollView>
    );
}
🔤
useLocalSearchParams ו-ScrollView
  • const { id } = useLocalSearchParams() — קורא את הפרמטר מה-URL הדינמי. בדיוק req.params.id מהרצאה 7, ו-useParams מהרצאה 9 — אותו רעיון, סביבה אחרת.
  • filter(...)[0] — מוצא את הכתבה לפי id (בדיוק כמו ה-Model מהרצאה 7!).
  • <ScrollView> — View שניתן לגלול (כתבה ארוכה לא נכנסת למסך אחד).

★ סיכום ההרצאה

🎯
התמונה המלאה
  • React Native = כותבים React, מקבלים אפליקציית מובייל native (cross-platform). Expo = ערכת הכלים שמקלה על הכל.
  • Core Components במקום HTML: View(=div), Text(=span, חובה לעטוף כל טקסט!), Image, Button, TextInput, ScrollView, FlatList, TouchableOpacity.
  • StyleSheet + Flexbox במקום CSS: אובייקטי JS, camelCase, flex: 1.
  • Expo Router — ניווט מבוסס-קבצים: Stack, Tabs, useRouter (push/back/replace), נתיב דינמי [id] + useLocalSearchParams.
  • FlatList לרשימות יעילות (במקום map), עם renderItem + keyExtractor.
  • אותם רעיונות מ-React: components, props, state, JSX — רק מבטא מובייל.
🔗
חיבורים זוהי בדיוק גישת ה-cross-platform מהרצאה 8 — אותו React, פלטפורמה אחרת. ה-components/props/state מהרצאה 8 חוזרים כולם. [id]+useLocalSearchParams הם ה-/:id מהרצאה 7 וה-useParams מהרצאה 9. onPress בלי סוגריים — שוב "מעבירים פונקציה, לא קוראים לה". ו-Foo News ממשיך ללוות אותנו — מ-XML, דרך Express, React ו-MongoDB, ועכשיו מובייל.
