1. מהי JavaScript — מאפיינים

🧭
קונטקסט לפני שכותבים קוד, חשוב להבין את האופי של השפה — מאפיינים שמסבירים למה היא מתנהגת כמו שהיא מתנהגת.
  • שפה אינטואיטיבית — אבל לא טריוויאלית (יש מלכודות עדינות).
  • רצה על הרבה פלטפורמות (דפדפן, שרת, מובייל...).
  • לא באמת קשורה ל-Java (השם הוא בעיקר שיווק מהשנים ההן).
  • רוב היסודות סטנדרטיים (if/else, לולאות וכו').
  • Weakly-typed (טיפוסים גמישים), interpreted (מפורשת, עם JIT-compilation).
  • Case sensitive — foo ≠ Foo.
  • Single-threaded, event-driven (thread אחד + event loop — בדיוק כמו שראינו בהרצאה 5).
  • Multi-paradigm — תומכת בפרוצדורלי, מונחה-עצמים, ופונקציונלי.

2. הקוויארקים — אל תנצל לרעה!

⚠️
"JavaScript can be weird, but DON'T abuse that" השפה מתירנית מאוד — אבל רק כי אפשר לעשות משהו לא אומר שצריך. הנה שלושה "היתרים" שעדיף להימנע מהם:
js · קוויארקים
// 1. נקודה-פסיק אופציונלית (אבל תמיד תכתוב אותה!)
foo = 1;
foo = 1

// 2. הצהרת משתנה אופציונלית (אבל תמיד הצהר!)
foo = 1;
var foo = 1

// 3. הצהרה כפולה — מותרת ולא זורקת שגיאה
var foo = 1;
var foo;
console.log(foo); // 1   (foo נשאר 1, לא נמחק!)
💡
הלקח תמיד כתוב נקודה-פסיק, תמיד הצהר משתנים (ועדיף עם let/const ולא var), ואל תצהיר אותו משתנה פעמיים. הסלחנות של השפה היא מקור עיקרי לבאגים — וכל החלק הזה של ההרצאה בא להראות איפה היא מסתתרת.

3. מנוע JS — Interpreted, JIT, V8

🧭
קונטקסט — איך הקוד באמת רץ השם הרשמי של JavaScript הוא ECMAScript, עם עדכונים שנתיים מאז 2015 (גרסת ES6 / ES2015 ואילך הביאו את רוב מה שנלמד היום). אבל איך הקוד מורץ?

JavaScript היא שפה מפורשת (interpreted), והמסלול הוא:

 קוד JS
   │ 1. ה-engine מנתח (parse) את הקוד לזיהוי שגיאות
   ▼      (באמצעות Abstract Syntax Tree — AST)
 AST
   │ 2. הומר ל-bytecode
   ▼
 Bytecode
   │ 3. מורץ ע"י JavaScript VM,
   ▼      ואפשר לייעל אותו עם JIT-compiler
 ריצה מהירה
  • JavaScript מורצת ע"י מנועים (engines) ייעודיים.
  • בעבר המנועים היו רק בדפדפנים (להרצת JS בצד הלקוח).
  • היום JS פופולרית גם בצד השרת — למשל מנוע V8 של Chrome משמש את Node.js.
🔤
Engine APIs — מה ייחודי לכל סביבה שפת הליבה זהה בכל המנועים, אבל לכל מנוע יכולה להיות פונקציונליות ייחודית שהיא חלק מ-API: console.log היא דוגמה אחת; alert(...) היא דוגמה אחרת — היא חלק מה-WebAPI של הדפדפן ולא קיימת ב-Node.js. כלומר, אותו קוד "core" ירוץ בכל מקום, אבל קריאות API מסוימות תלויות-סביבה.

4. Hello World + הפרדת HTML/JS

JavaScript משובצת בדף HTML בתוך תג <script>:

html + js · hello world
<html>
<body>
    <script type="text/javascript">
        var x = 'Hello world';
        alert(x);
    </script>
</body>
</html>

alert(x) מציג תיבת הודעה בדפדפן עם הטקסט. (שים לב: רק קוד בתוך <script> מורץ — טקסט מחוץ לו הוא סתם תוכן הדף.)

עדיף להפריד את ה-JS לקובץ נפרד (כמו שמפרידים CSS) — קריא יותר וניתן למחזור:

html · script חיצוני
<html>
<head></head>
<body>
    <script type="text/javascript" src="foo.js"></script>
</body>
</html>
🔗
חיבור להרצאה 2 זו בדיוק הפרדת אחריות (separation of concerns) שראינו ב-SOLID: HTML = מבנה, CSS = עיצוב, JS = התנהגות. ערבוב של שלושתם באותו קובץ הוא code smell.

חלק ב' · DOM, אירועים, jQuery

5. DOM — מודל המסמך (Document Object Model)

🧭
קונטקסט איך JS "נוגעת" בדף שמוצג? דפדפנים מציגים מסמכי web. כשהדפדפן טוען מסמך, הוא יוצר ייצוג אובייקטי שלו — זהו ה-DOM. ל-DOM יש API שמאפשר לקרוא, לשנות ולתמרן את הדף בזמן ריצה.

האובייקט הגלובלי document נותן גישה לחלקי הדף:

js · ממשק document
console.log(document.head);      // אלמנט ה-head
console.log(document.body);      // אלמנט ה-body
console.log(document.forms);     // כל הטפסים בדף
console.log(document.links);     // כל הקישורים (<a>)
console.log(document.images);    // כל התמונות (<img>)
console.log(document.cookie);    // העוגיות
console.log(document.location);  // ה-URL הנוכחי

כל אחד מהם מחזיר אוסף או אובייקט שמייצג חלק מהדף — שאפשר לקרוא ולשנות.

6. DOM + Events

🧭
קונטקסט זוכרים ש-JS היא event-driven? הדרך העיקרית להגיב למשתמש היא אירועים (events): לחיצה, הקלדה, מעבר עכבר... מאתרים אלמנט ב-DOM ומחברים לו פונקציית טיפול (handler).

דוגמה 1 — כפתור עם onclick:

html
<button id="btnFoo">Click me</button>
<script type="text/javascript" src="index.js"></script>
js · index.js
function show(){
    alert("You clicked me!");
}

var btnFoo = document.getElementById('btnFoo');
btnFoo.onclick = show;   // שים לב: show בלי () — מעבירים את הפונקציה, לא קוראים לה!
⚠️
מלכודת נפוצה — show מול show() כותבים btnFoo.onclick = show (בלי סוגריים!) — מעבירים הפניה לפונקציה שתופעל בעת הלחיצה. אם תכתוב show() עם סוגריים, הפונקציה תרוץ מיד (בעת טעינת הדף) ותשמור את ערך ההחזרה שלה כ-handler — באג קלאסי. בדיוק כמו start() מול run() ב-Java threads מהרצאה 4!

דוגמה 2 — תגובה להקלדה (onkeyup) + עדכון innerHTML:

js · echo להקלדה
function handleKeyup(){
    var divBar = document.getElementById('divBar');
    divBar.innerHTML = '<u>' + inpFoo.value + '</u>';
}

var inpFoo = document.getElementById('inpFoo');
inpFoo.onkeyup = handleKeyup;

בכל הקלדה, התוכן של ה-div מתעדכן לערך השדה (מקווקו). innerHTML מאפשר לכתוב HTML חדש לתוך אלמנט.

דוגמה 3 — צביעת מספר אלמנטים + this:

js · mouseover/mouseout
function handleMouseover(){
    this.style.backgroundColor = 'blue';
}
function handleMouseout(){
    this.style.backgroundColor = 'red';
}

var divs = document.getElementsByTagName('div');
for (let i = 0; i < divs.length; i++) {
    divs[i].onmouseover = handleMouseover;
    divs[i].onmouseout = handleMouseout;
}
🔤
מהו this כאן בתוך handler, this מצביע על האלמנט שעליו קרה האירוע. לכן אותה פונקציה אחת (handleMouseover) עובדת על כל ה-divs — כל אחד צובע את עצמו. שים לב גם ל-let i בלולאה (ולא var) — נראה מיד למה זה קריטי.

דרכים נוספות ליצור/לאתר אלמנטים:

js · create / find
document.write("Hello world");
document.write("<h1>Hello world</h1>");

var txt = document.createTextNode(document.body.innerHTML);
var div = document.createElement("div");   // יצירת אלמנט חדש
div.appendChild(txt);                       // הוספת ילד
document.body.appendChild(div);             // הוספה לדף

const byId      = document.getElementById("foo");
const byTagName = document.getElementsByTagName("bar");
const byClass   = document.getElementsByClassName("baz");

חיבור אירועים — שתי דרכים: ישירות ב-HTML (inline) או מ-JS:

html · inline events
<body onClick="console.log('click')" onLoad="console.log('load')">
    <input onFocus="console.log('focus')"
           onBlur="console.log('blur')"
           onChange="console.log('change')">
</body>
אירועמתי
onLoadהדף סיים להיטען.
onClickלחיצה.
onFocus / onBlurשדה קיבל / איבד מיקוד.
onChangeערך השדה השתנה (אחרי איבוד מיקוד).
onSubmitטופס נשלח (אם מחזיר false — השליחה מבוטלת).

7. jQuery — ספריית JavaScript

🧭
קונטקסט ה-DOM API הגולמי מסורבל (document.getElementById וכו'). jQuery היא ספרייה שמפשטת בחירת אלמנטים, מניפולציה ואירועים, עם הפונקציה הקסומה $.
js · jQuery בסיסי
$(document).ready(function () {
    console.log("Hello world");   // רץ כשה-DOM מוכן
});

בחירה ומניפולציה — סלקטורים בסגנון CSS:

js · סלקטורים
$(document).ready(function () {
    var a = $('div');              // כל ה-div-ים
    console.log(a.length);

    $('.bar').each(function(i, elem){   // לולאה על מחלקה .bar
        console.log($(elem).text());
    });

    var c = $('#foo');             // לפי id
    c.attr('x', 'y');              // הוספת attribute

    var d = $('div div.bar');      // סלקטור מקונן
    d.addClass('zzzz');            // הוספת class
});

אירועים מתקדמים — שרשור, מניעת התנהגות ברירת מחדל, קודי מקשים:

js · jQuery events
$(function () {
    $('body').click(function(){
        $('#foo').toggle();             // הצג/הסתר
    });

    $('input#search').click(function(event){
        event.stopPropagation();        // עצור "בעבוע" האירוע למעלה
    }).keydown(function(event){
        if (event.keyCode == 13 && this.value.length < 5){  // Enter = 13
            $('#foo div').text('5 letters minimum');
            event.preventDefault();     // בטל את התנהגות ברירת המחדל
        }
    }).keyup(function(event){
        if (event.keyCode == 13) return;
        $('#foo div').text($(this).val());
    });
});
🔤
מושגי מפתח
  • $('selector') — בוחר אלמנטים בסגנון CSS (#id, .class, tag).
  • Chaining — .click().keydown().keyup() משורשרים על אותו אובייקט.
  • event.stopPropagation() — מונע מהאירוע "לבעבע" (bubble) להורים.
  • event.preventDefault() — מבטל את הפעולה הדיפולטית (למשל שליחת טופס).
  • event.keyCode == 13 — קוד המקש Enter.

8. BOM — Browser Object Model

🧭
קונטקסט ה-DOM מאפשר להתממשק עם הדף; באופן דומה, ה-BOM מאפשר להתממשק עם הדפדפן עצמו. האובייקט הגלובלי הוא window — ולמעשה document הוא window.document, וכל משתנה גלובלי (window.foo) הוא תכונה של window.
js · window
window.document    // אותו document שהכרנו
window.foo         // undefined  (עוד לא הוגדר)
var foo = {a:1}
window.foo         // {a: 1}     (משתנה גלובלי = תכונה של window)

חלק ג' · שפת הליבה לעומק

9. var / let / const ו-scoping

🧭
קונטקסט — אחד הנושאים החשובים ביותר לפני ES6 הייתה רק var, וההתנהגות שלה מבלבלת. ES6 הוסיפה את let ו-const שמתנהגות צפוי יותר. ההבדל המרכזי: scope (תחום הקיום).

var אינו block-scoped — בלוק if לא יוצר scope חדש:

js · var בתוך if
var x = 1;
console.log(x); // 1

if (x == "1"){       // true! (== ממיר טיפוסים; === היה מחזיר false)
    var x = 2;       // אותו x! לא חדש
    console.log(x);  // 2
}

console.log(x); // 2   ← השתנה! ה-x בשורה 1 ובתוך ה-if הם אותו משתנה

var כן function-scoped — פונקציה כן יוצרת scope:

js · var בתוך function
var x = 1;
function foo() {
    var x = 2;       // x נפרד! (scope של הפונקציה)
    console.log(x);  // 2
}
foo();
console.log(x); // 1   ← לא השתנה

let הוא block-scoped — כל בלוק {} יוצר scope:

js · let
let x = 1;
console.log(x); // 1

if (x == "1"){
    let x = 2;       // x חדש! (scope של הבלוק)
    console.log(x);  // 2
}

function foo() {
    let x = 3;       // x חדש נוסף
    console.log(x);  // 3
}
foo();

console.log(x); // 1   ← לא הושפע משום בלוק/פונקציה

const — כמו let אבל לא ניתן להשמה מחדש:

js · const
const foo = 1;
foo = 1;  // Uncaught TypeError: Assignment to constant variable.
🌍
מלכודת — משתנים לא-מוצהרים הם גלובליים! אם משתמשים במשתנה בלי להצהיר אותו בכלל, JS מצהירה אותו אוטומטית כגלובלי:
js
function foo() {
    x = 2;            // x לא מוצהר → הופך לגלובלי!
    console.log(x);   // 2
}
foo();
console.log(x); // 2   ← נגיש מבחוץ! (זיהום ה-global scope)
זו אחת הסיבות המרכזיות לכתוב תמיד let/const ולהפעיל "use strict".
💡
כלל אצבע (דוגמה משלי) ב-JS מודרני: השתמש ב-const כברירת מחדל, ב-let רק כשצריך לשנות את הערך, וב-var אף פעם. זה מונע את רוב המלכודות לעיל. זוכרים את לולאת ה-let i מקטע האירועים? עם var i כל ה-handlers היו "רואים" את אותו i הסופי — באג קלאסי שבדיוק let פותר.

10. אובייקטים (JSON) ומתודות

🧭
קונטקסט אובייקטים ב-JS הם פשוט אוספי מפתח-ערך, בתחביר שנקרא JSON (JavaScript Object Notation). הם דינמיים — אפשר להוסיף ולשנות שדות תוך כדי ריצה.
js · אובייקט דינמי
const o = {foo: "Bar", caz: 3, bar: 'abc'}
console.log(o);   // {foo: 'Bar', caz: 3, bar: 'abc'}

o.caz = 4;        // שינוי שדה קיים
console.log(o);   // {foo: 'Bar', caz: 4, bar: 'abc'}

o.xyz = '5';      // הוספת שדה חדש תוך כדי ריצה!
console.log(o);   // {foo: 'Bar', caz: 4, bar: 'abc', xyz: '5'}

o = {}            // TypeError! (const — אי-אפשר להשמה מחדש)
🔤
const על אובייקט — נקודה עדינה const o מונע השמה מחדש של o (o = {} אסור), אבל כן מאפשר לשנות את תוכן האובייקט (o.caz = 4 מותר). ה-const "נועל" את ההפניה, לא את התוכן.

מתודות באובייקט + הגישה אליהן:

js · מתודות
const o = {
    foo: 1,
    caz() { return this.foo },           // קיצור מתודה
    //caz2: () => { return this.foo },   // ❌ arrow — אין לה this נכון!
    bar: function(x) { return x+1 }
}
console.log(o.foo)      // 1
console.log(o['foo'])   // 1   (גישה גם בסוגריים מרובעים)
console.log(o.caz())    // 1
console.log(o.bar(1))   // 2
⚠️
אל תשתמש ב-arrow functions כמתודות! ל-arrow functions יש מגבלות מובנות — אין להן גישה ל-this של האובייקט (ה-this שלהן נקבע מההקשר החיצוני). לכן caz2 המקוונן לא היה עובד. למתודה שצריכה this — השתמש ב-caz() או ב-function רגיל.

11. Hoisting (הרמה)

🧭
קונטקסט לפני שהקוד רץ, ה-engine סורק אותו ו"מרים" (hoists) את ההצהרות לראש ה-scope שלהן. חשוב: מורמות רק ההצהרות (declarations), לא האתחול (initialization).

עם var — הצהרה בלי אתחול = undefined (לא שגיאה):

js · hoisting של var
// זה כל הקוד:
console.log(foo); // undefined   ← לא שגיאה! ההצהרה "הורמה" למעלה
var foo;

// מול: משתנה שלא קיים בכלל
console.log(bar); // ReferenceError: bar is not defined

Hoisting מרים את ההצהרה, לא את הערך:

js
console.log(foo); // undefined   ← ההצהרה הורמה, אבל לא ה-='bar'
var foo = 'bar';
console.log(foo); // bar
🪤
let ו-const — Temporal Dead Zone ל-let/const ההתנהגות שונה — גישה לפני ההצהרה זורקת שגיאה (הם ב-"אזור מת זמני", TDZ):
js
console.log(foo); // ReferenceError: foo is not defined
let foo = 'foo';
זה התנהגות בריאה יותר — עוד סיבה להעדיף let/const על var.

הצהרת פונקציה מול ביטוי פונקציה — הבדל קריטי:

js · function declaration — עובד
foo(); // foo   ← עובד! הצהרת פונקציה מורמת במלואה
function foo(){
    console.log("foo");
}
js · function expression — נכשל
foo(); // TypeError: foo is not a function
var foo = function (){          // רק ההצהרה var foo מורמת (=undefined),
    console.log("foo");         // ההשמה של הפונקציה עוד לא קרתה!
}
🔤
ההבדל הצהרת פונקציה (function foo(){}) מורמת במלואה — אפשר לקרוא לה לפני ההגדרה. ביטוי פונקציה (var foo = function(){}) — רק ה-var foo מורם (כ-undefined), וקריאה לפני ההשמה נותנת "foo is not a function". וגם בתוך פונקציה: var פנימי מורם לראש הפונקציה, ולכן console.log(foo) לפני var foo='foo' יחזיר undefined ולא את הערך החיצוני.

12. Modules

🧭
קונטקסט איך מארגנים קוד גדול? Module, בדומה ל-class, מקבץ יחד נתונים ופעולות. אבל בניגוד ל-class (שבו הכל public) — module חושף רק את מה שהוא מחזיר/מייצא; השאר נשאר פרטי.

ES Modules — קובץ אחד = module אחד. מה שexport-ים זמין לאחרים; מה שלא — פרטי:

js · Foo.js — ייצוא
// דרך 1 — export ישיר
export function Foo(bar) {
    // ...
}

// דרך 2 — export בנפרד
function Foo(bar) {
    // ...
}
export Foo;
js · קובץ אחר — ייבוא
import Foo from "Foo.js"

var foo = Foo(1)
🔗
כבר ראינו את זה! בהרצאה 5, ב-MVC של NodeJS, השתמשנו ב-module.exports / require (סגנון CommonJS). זו אותה רעיון — אנקפסולציה ברמת הקובץ: חושפים API מינימלי ומסתירים את הפרטים, בדיוק כמו ב-encapsulation של OOP.

13. Async: Callbacks, Promises, async/await

🧭
קונטקסט ראינו בהרצאה 5 את ה-event loop, Promises ו-async/await. כאן נחזק את ההבנה דרך הזווית של ההיסטוריה: callbacks → Promises → async/await, וננתח דוגמת סדר-ריצה מדויקת.

הדרך הישנה — Callbacks: מעבירים פונקציה שתורץ "כשיהיה מוכן":

js · callback עם XMLHttpRequest
function foo(callback){
    const xhr = new XMLHttpRequest();
    xhr.onload = function() {
        callback(xhr.responseText)   // קוראים ל-callback כשהתשובה הגיעה
    }
    xhr.open("GET", "http://www.foo.com");
    xhr.send();
}

function bar(data) {
    console.log(data)
}

foo(bar);   // מעבירים את bar כ-callback

Event loop — תזכורת: קוד סינכרוני רץ על ה-Stack, פעולות אסינכרוניות עוברות ל-APIs, וה-callbacks שלהן נכנסים ל-Task queue. ה-event loop מעביר אותם ל-Stack רק כשהוא ריק. לכן:

js · מה קודם?
setTimeout(function (){ console.log('foo'); }, 2000);
console.log('bar');
// פלט: bar ואז foo

דוגמת Promise מלאה — נתח את סדר הפלט:

js · סדר ריצה
console.log(1)
let foo = new Promise((resolve, reject) => {
    console.log(2)
    setTimeout(function() {
        console.log(3)
        resolve('done');
    }, 3000);
    console.log(4)
})

console.log(7)

function bar(data) {
    console.log(6)
    console.log(data)
}

console.log(8)
foo.then(bar);
console.log(9)
🔢
הפלט: 1, 2, 4, 7, 8, 9, 3, 6, done — למה?
  • קוד סינכרוני קודם: 1. ה-Promise executor רץ מיד וסינכרונית: 2, מתזמן את ה-setTimeout, 4. ממשיכים: 7, 8, רושמים את foo.then(bar), 9.
  • אחרי 3 שניות (כשה-Stack ריק): ה-callback של ה-timeout רץ → 3, ואז resolve('done') מפעיל את bar → 6, ואז console.log(data) → done.
השורה המספרית (3000ms) רצה אחרונה — בדיוק כמו ש-fetch(...).then(...) רץ אחרי הקוד הסינכרוני שאחריו.

async/await — פונקציות async תמיד מחזירות Promise:

js · async/await
let foo = async () => "bar";
// זהה ל: function foo() { return Promise.resolve("bar"); }

var bar = await foo()   // await "פותח" את ה-Promise ומחזיר את הערך
console.log(bar)        // bar

await ממתין שה-Promise ייפתר ומחזיר את הערך — קוד אסינכרוני שנראה סינכרוני וקריא (ראינו בהרצאה 5).

14. Prototypes ושרשרת הפרוטוטייפ

🧭
קונטקסט — מה באמת קורה מאחורי class ל-JavaScript יש class ו-extends כמו ב-Java — אבל זה הטעיה. מתחת למכסה המנוע, JS היא prototype-based, וה-class הוא רק "סוכר תחבירי" (syntactic sugar) מעל מנגנון ה-prototypes.
js · class ו-extends
class Foo {
    caz = 1
    constructor (foo) {
        this.bar = foo
    }
    baz() {
        return this.bar
    }
}

class Foo2 extends Foo {
    constructor(foo2) {
        super(foo2)
    }
}

const f = new Foo2(3)
console.log(f)   // Foo2 {caz: 1, bar: 3}, [[Prototype]]: Foo

איך זה עובד באמת:

  • לכל אובייקט יש תכונה מיוחדת — prototype.
  • ערך ה-prototype הוא גם אובייקט, ולכן גם לו יש prototype — וכך נוצרת שרשרת פרוטוטייפ (prototype chain).
  • כשמחפשים תכונה של אובייקט: מחפשים קודם באובייקט עצמו, ואם לא נמצא — מחפשים ב-prototype שלו, וחוזרים על זה עד שמגיעים ל-null.
 f (Foo2)  ──prototype──▶  Foo.prototype  ──prototype──▶  Object.prototype  ──▶  null
 {caz, bar}                {baz, constructor}            {toString, valueOf, ...}

 f.baz()  →  לא ב-f? → חפש ב-Foo.prototype → נמצא! ✓
 f.toString()  →  לא ב-f? → לא ב-Foo? → ב-Object.prototype → נמצא! ✓
🔗
בדיקה בקונסול Object.getPrototypeOf(c) מחזיר את ה-prototype של אובייקט. אם תמשיך לטפס (getPrototypeOf(getPrototypeOf(c))) תגיע בסוף ל-null — קצה השרשרת. זה בדיוק מה ש-JS עושה בכל גישה לתכונה. הסיכום: JavaScript היא prototype-based; ה-class רק נראה כמו ירושה קלאסית אבל מתורגם לשרשרת prototypes.

חלק ד' · מסגרות מודרניות

15. React ו-Transpilation

🧭
קונטקסט לתמרן את ה-DOM ידנית (createElement, appendChild) זה מסורבל ומועד לשגיאות. React היא הספרייה הפופולרית שמפשטת בניית ממשקים. נראה את ההתפתחות צעד-צעד.

DOM גולמי — איך עושים את זה בלי React:

js · DOM ידני
const h = document.createElement('h1')
h.textContent = 'Hello'
h.className = 'foo'
document.getElementById('root').appendChild(h)

אותו דבר עם React — React.createElement + ReactDOM.render:

js · React.createElement
const h = React.createElement('h1', { className: 'foo' }, 'Hello')
ReactDOM.render(h, document.getElementById('root'))

JSX — כותבים "HTML" בתוך JS (קריא בהרבה מ-createElement):

jsx · JSX
const h = <h1>Hello {1+2}
    <div>
        <span className="foo" foo="bar">Hi</span>
    </div>
</h1>

ReactDOM.render(h, document.getElementById('root'))

שים לב ל-{1+2} — אפשר להזריק ביטויי JS לתוך ה-JSX, ול-className במקום class (כי class שמורה ב-JS).

קומפוננטות + props — פונקציה שמחזירה JSX, עם פרמטרים:

jsx · component
function Foo({a, b, c}) {       // destructuring של props
    return (
        <div>
            a: {a}, b: {b}, c: {c}
        </div>
    );
}

function App() {
    var i = 1+2;
    return (
        <div>
            <Foo a="1" b="a" c={i} />
            <Foo a="2" b="b" c={i+1} />
            <Foo a="3" b="c" c={i+i} />
        </div>
    );
}

State עם useState — קומפוננטה ש"זוכרת" מצב ומתעדכנת:

jsx · useState
import { useState } from 'react';

function Foo() {
    const [bar, setBar] = useState(0);   // [ערך, פונקציית-עדכון]
    console.log(bar);
    return (
        <div onClick={() => setBar(bar + 1)}>
            {bar}
        </div>
    );
}

בכל לחיצה setBar מעדכן את ה-state, ו-React מרנדר מחדש את הקומפוננטה אוטומטית עם הערך החדש.

🔤
Transpilation — איך JSX/ES חדש רץ בדפדפנים ישנים JavaScript מתפתחת ומוסיפה תחביר חדש. יכול להיות שתכתוב קוד תקין לפי ES החדש, אבל דפדפן ישן לא תומך בו. Transpilation פותר זאת — ממיר תחביר חדש לתחביר ישן. Babel הוא transpiler כזה: הוא ממיר JSX ו-ES6+ ל-JS שדפדפנים ישנים מבינים. (בדף אפשר לטעון babel.js ולסמן <script type="text/babel">.)
⚛️
create-react-app — פרויקט אמיתי בפועל לא טוענים React מ-CDN אלא יוצרים פרויקט: npx create-react-app hello_world. זה מקים מבנה מוכן (src/App.js, public/index.html...) עם פקודות: npm start (שרת פיתוח), npm run build (חבילה לפרודקשן), npm test (בדיקות). React מטפל ב-transpilation, bundling וכו' מאחורי הקלעים.

★ סיכום ההרצאה

🎯
התמונה המלאה
  • JS: weakly-typed, interpreted (JIT, V8/Node), case-sensitive, single-threaded event-driven. סלחנית — אבל אל תנצל זאת לרעה.
  • DOM = ייצוג אובייקטי של הדף (document); BOM = הדפדפן (window). מגיבים למשתמש דרך events (זכור: onclick = show בלי סוגריים, this = האלמנט).
  • jQuery: $('selector'), chaining, stopPropagation/preventDefault.
  • scoping: var = function-scoped + hoisted ל-undefined; let/const = block-scoped + TDZ. השתמש ב-const/let, לעולם לא var. משתנה לא-מוצהר = גלובלי (סכנה!).
  • Hoisting: הצהרות עולות לראש ה-scope (לא האתחול). function declaration עולה במלואה; function expression לא.
  • אובייקטים דינמיים; אל תשתמש ב-arrow כמתודה (אין this). Modules: חושפים רק מה ש-export.
  • Async: callbacks → Promises → async/await; הכל מעל event loop יחיד.
  • Prototypes: JS היא prototype-based; class הוא syntactic sugar מעל prototype chain.
  • React: createElement → JSX → components + props + useState; Babel ל-transpilation.
